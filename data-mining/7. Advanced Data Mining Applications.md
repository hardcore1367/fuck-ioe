# 7. Advanced Data Mining Applications

## 1. Multimedia Mining

### 1.1 What is Multimedia Mining
A multimedia database stores and manages large collections of multimedia data such as audio, video, images, graphics, and speech. **Multimedia/image mining** deals with extracting implicit knowledge, data relationships, or patterns that are not explicitly stored within the images/multimedia itself.

### 1.2 Fundamental Challenge
The key challenge is determining how the **low-level pixel representation** in an image (or image sequence) can be effectively and efficiently processed to identify **high-level spatial objects and relationships**.

### 1.3 Process and Techniques
Typical multimedia mining involves: preprocessing → transformation and feature extraction → mining → evaluation and interpretation of the knowledge found. Standard data mining techniques such as **association rules** and **clustering** can be applied on the extracted features.


## 2. Web Mining

### 2.1 What is Web Mining
Web mining is the application of data mining techniques to extract knowledge from Web data — namely **Web content**, **Web structure**, and **Web usage** data.

### 2.2 Web Mining Taxonomy
Web mining is broadly divided into three categories based on the kind of data mined:

**a. Web Content Mining**
- Process of extracting useful information from the contents of Web pages/documents.
- Content includes text, images, audio, video, or structured records like lists and tables.
- Most widely researched area; issues include topic discovery, extracting association patterns, clustering of Web documents, and classification of Web pages.

**b. Web Structure Mining**
- Discovers structure information from the Web, where a Web graph has pages as nodes and hyperlinks as edges.
- Two types:
  - **Hyperlinks:** a structural unit connecting one location to another — either within the same page (**intra-document**) or between different pages (**inter-document**).
  - **Document structure:** the content within a page organized in tree form based on HTML/XML tags; mining here focuses on extracting the document object model (DOM) structure.

**c. Web Usage Mining**
- Applies data mining to discover interesting usage patterns from Web data, to better understand and serve Web-based applications.
- Captures the identity/origin of users along with their browsing behavior.
- Classified by kind of usage data:
  - **Web server data:** logs collected by the server (IP address, page reference, access time).
  - **Application server data:** logs generated by commercial application/e-commerce servers tracking business events.
  - **Application level data:** custom-defined events logged by the application itself.

### 2.3 Challenges of Web Mining
- Too huge in volume for effective data warehousing and mining.
- Too complex and heterogeneous in structure/content.
- Growing and changing rapidly.
- Broad diversity of user communities.
- Only a small portion of the information on the Web is truly relevant or useful.

### 2.4 PageRank Algorithm
PageRank, described by Lawrence Page and Sergey Brin, ranks Web pages based on the link structure of the Web (part of Web structure mining):
$$PR(A) = (1-d) + d \left(\frac{PR(T_1)}{C(T_1)} + \dots + \frac{PR(T_n)}{C(T_n)}\right)$$
Where:
- $PR(A)$ = PageRank of page A
- $PR(T_i)$ = PageRank of pages $T_i$ that link to page A
- $C(T_i)$ = number of outbound links on page $T_i$
- $d$ = damping factor (typically 0.85), set between 0 and 1

**Key points:**
- PageRank is computed for each page individually, not for a whole website.
- It is **recursively defined** — the PageRank of page A depends on the PageRank of the pages linking to it.
- A page's contribution is **weighted by its number of outbound links** — the more outbound links a page has, the less benefit it passes on to any single page it links to.
- An additional inbound link to a page will always increase that page's PageRank.
- A second version of the formula, $PR(A) = \frac{1-d}{N} + d(\dots)$, normalizes by the total number of pages N on the web.
- Because the actual Web has billions of pages, an exact solution is impractical, so PageRank values are computed using an **iterative approximation** — each page starts with an initial value, and PageRank is recalculated over several iterations until the values stabilize (converge).


## 3. Time Series Data Mining

### 3.1 What is Time Series Data Mining
A time series consists of a sequence of values or events collected over repeated measurements of time, usually at equal time intervals. It is used in applications like stock prediction and economic analysis. Time series analysis has two main goals:
- **Modeling time series:** generating the time series with its underlying generating mechanism.
- **Forecasting time series:** predicting future values of the time series variables.

### 3.2 Components of Time Series (Trend Analysis)
- **Trend / Long-term movements:** the general direction in which the series moves over a long or short interval, shown via a trend curve or trend line.
- **Cyclic movement:** long-term oscillations about the trend curve/line, which may or may not be periodic.
- **Seasonal movements (Seasonality):** systematic, calendar-related variations that repeat at regular intervals (e.g., a sudden rise in sales of sweets during Tihar). Unlike cyclic variation, seasonal variation has a fixed, known period (daily, monthly, yearly).
- **Irregular / Random movements:** variation caused by random or chance events (e.g., a price rise due to a sudden supply crisis).

### 3.3 Approaches for Time Series Analysis
- **Regression analysis:** commonly used to find the trend in time series data.
- **Seasonal index:** used to adjust the relative values of a variable to account for seasonal effects.
- **Autocorrelation analysis:** correlation between the $i^{th}$ element of the series and the $(i-k)^{th}$ element, used to detect seasonal patterns, where k is called the "lag".
- **Moving average:** a common method for determining trend — computed by averaging a fixed-size window of consecutive values (e.g., moving average of order 3 averages every 3 consecutive points) and sliding the window across the series. A **weighted moving average** gives different weights to different points in the window instead of an equal average.
- **Freehand method:** drawing an approximate curve/line to fit the data based on the user's own judgment.
- **Least squares method:** used to mathematically fit the best-fitting curve/line to the data.
