
## 1. Introduction

Real-world data is often incomplete, noisy, and inconsistent, which can lead to incorrect results if used directly. Data preprocessing in data mining is the process of cleaning and preparing raw data so it can be used effectively for analysis and model building.

## 2. Data Types and Attributes

An **attribute** (or feature/variable) is a data field representing a characteristic of a data object.

### 2.1 Attribute Types

#### 2.1.1 Approach 1: By Measurement Scale

- **Nominal** — values are just different names or labels; they only support equality checks (=, ≠). No ordering or arithmetic is meaningful. Examples: zip codes, employee IDs, eye color, blood type.
- **Ordinal** — values have a meaningful order or ranking among them, but the actual magnitude of the gap between successive values is not known. Support comparison operators (<, >) but not arithmetic. Examples: hardness of minerals, {good, better, best}, grades, street numbers.
- **Interval** — differences between values are meaningful because a fixed unit of measurement exists. Support addition and subtraction (+, −) but not multiplication or division, since there is no true zero point. Examples: calendar dates, temperature in °C or °F (0°C does not mean "no temperature").
- **Ratio** — both differences and ratios are meaningful; a true zero point exists, meaning zero indicates a complete absence of the quantity. Support all arithmetic operations (×, ÷). Examples: temperature in Kelvin, age, mass, length, monetary quantities, counts.

#### 2.1.2 Approach 2: By Number of Values

- **Discrete attribute** — has only a finite or countably infinite set of values, meaning you can list or count all possible values. Often represented as integer variables. Binary attributes are a special case of discrete attributes. Examples: zip codes, counts, the set of words in a document collection.
- **Continuous attribute** — has real numbers as attribute values and can theoretically take any value within a range. In practice, real values can only be measured and represented using a finite number of digits, so continuous attributes are typically stored as floating-point variables. Examples: temperature, height, weight.

#### 2.1.3 Approach 3: By Representation

- **Character** — values are represented in the form of a character or a set of characters (string). These are qualitative in nature and cannot be used directly in arithmetic operations. Examples: names, labels, codes, descriptions.
- **Number** — values are represented in numeric form, either as whole numbers (integers) or decimal numbers (real/floating-point). These support mathematical operations and are used in quantitative analysis.

### 2.2 Symmetric vs Asymmetric Binary Attributes

A **symmetric binary attribute** is one where both states (0 and 1) are equally important and interchangeable. There is no preference on which outcome is coded 0 or 1.

- Example: *gender* (male/female) — neither value is more significant.

An **asymmetric binary attribute** is one where the two outcomes are not equally important. By convention, the more important (usually rarer) outcome is coded 1.

- Example: *HIV test result* — positive (1) is far more significant than negative (0).

### 2.3 Dimensionality

**Dimensionality** refers to the number of attributes (features) in a dataset. A **data dimension** is a set of data attributes pertaining to something of interest — things like "customers", "products", "stores", and "time". Higher dimensionality means each data object is described by more attributes.

### 2.4 Curse of Dimensionality

The **curse of dimensionality** refers to the phenomenon where, as the number of dimensions increases, data becomes increasingly sparse in the space it occupies.

- Definitions of **density and distance between points** become less meaningful — any two points appear roughly equidistant from each other, which breaks distance-based methods like clustering and outlier detection.
- The volume of the feature space grows exponentially, so you need exponentially more data to maintain the same density.
- Many features become irrelevant or redundant, acting as noise and degrading model accuracy.
- Computation and storage costs increase significantly.

### 2.5 Dimensionality Reduction

**Dimensionality reduction** reduces the number of features while retaining as much important information as possible.

**Purpose:**
- Avoid the curse of dimensionality.
- Reduce the amount of time and memory required by data mining algorithms.
- Allow data to be more easily visualized.
- Help eliminate irrelevant features or reduce noise.

**Techniques:**
- **Principal Component Analysis (PCA)** — transforms original correlated attributes into a smaller set of uncorrelated principal components ordered by variance captured.
- **Singular Value Decomposition (SVD)** — matrix factorization technique closely related to PCA; decomposes a data matrix into singular vectors and values to identify the most important structure.
- **Others** — supervised techniques (use class labels to guide reduction), non-linear techniques such as t-SNE and autoencoders (handle non-linear relationships that PCA cannot).

### 2.6 PCA — Steps

1. Normalize the data (mean = 0).
2. Compute the **covariance matrix** of the attributes.
3. Find the **eigenvectors and eigenvalues** of the covariance matrix.
4. Sort eigenvectors by eigenvalue descending — higher eigenvalue = more variance captured.
5. Select the top-k eigenvectors as the new axes.
6. Project the original data onto these k eigenvectors → reduced dataset.

PCs are orthogonal (uncorrelated) to each other. PCA is unsupervised — it uses no class label information.

### 2.7 Data Quality

**Data quality** refers to how well data satisfies the requirements of its intended use. The key dimensions are:

| Dimension | Meaning |
|---|---|
| **Accuracy** | Data values are correct and free of errors |
| **Completeness** | All required data is present (no missing records/values) |
| **Consistency** | Same data stored across sources agrees with itself |
| **Timeliness** | Data is up-to-date for the intended use |
| **Believability** | Data is trusted by users |
| **Interpretability** | Data is understandable to users |

Poor data quality leads to unreliable mining results — "garbage in, garbage out."

### 2.8 Data Preprocessing Tasks

The four major tasks in data preprocessing are:

#### 2.8.1 Data Cleaning

**Data cleaning** (also called data cleansing) is the process of detecting and correcting corrupt, inaccurate, or irrelevant records from a dataset. Real-world data is often dirty — it may contain missing values, duplicate entries, outliers, noise, or inconsistencies arising from human error, faulty sensors, or merging data from different sources. Data cleaning improves data quality so that subsequent mining produces reliable results. It addresses three main problems:

**Missing values** — a value may be absent because it was not recorded, not applicable, or unknown. Ways to handle:
- Ignore the tuple (only suitable if the class label itself is missing).
- Fill in manually (impractical for large datasets).
- Replace with a global constant such as "Unknown" or "N/A".
- Replace with the **attribute mean** (for normally distributed data) or **attribute median** (for skewed data).
- Use the most probable value via inference (e.g., regression or Bayesian methods).

**Noisy data** — random error or variance in measured values caused by faulty instruments, transmission errors, or human error. Handled by binning, regression, or clustering.

**Bin Smoothing (Binning)** — a technique to smooth noisy data by sorting values into bins and replacing each value with a smoothed estimate.

Steps:
1. Sort the data values.
2. Partition sorted values into bins of equal frequency (same number of values per bin).
3. Smooth values within each bin using one of:
   - **Smoothing by bin mean** — replace each value in the bin with the mean of the bin.
   - **Smoothing by bin median** — replace each value with the median of the bin.
   - **Smoothing by bin boundary** — replace each value with the nearest bin boundary (min or max).

**Example — Smoothing by bin means:**

Sorted data: 4, 8, 15, 21, 21, 24, 25, 28, 34

Partition into 3 equal-frequency bins:
- Bin 1: 4, 8, 15 → Mean = 9 → replaced as **9, 9, 9**
- Bin 2: 21, 21, 24 → Mean = 22 → replaced as **22, 22, 22**
- Bin 3: 25, 28, 34 → Mean = 29 → replaced as **29, 29, 29**

Wider bins produce smoother results. Binning also serves as a discretization technique.

**Inconsistencies** — same entity has conflicting values across sources (e.g., two different birthdates for the same customer, or "Age" stored as text in one source and integer in another). Resolved using domain knowledge, metadata, or integrity constraints.

#### 2.8.2 Data Integration

**Data integration** is the process of merging data from multiple heterogeneous sources — such as databases, flat files, or data cubes — into a single, coherent data store. In real-world scenarios, an organization's data is often spread across different systems that may use different formats, naming conventions, or encodings for the same real-world entities. Data integration brings all this data together so it can be analyzed as a whole. Key steps and challenges:
- **Schema matching** — identify attributes across sources that refer to the same real-world entity (e.g., "cust_id" vs "customer_no").
- **Redundancy detection** — an attribute derivable from another (e.g., annual_salary = monthly_salary × 12) is redundant and should be removed. Correlation analysis (chi-square for nominal, Pearson's r for numeric) helps detect this.
- **Data conflict resolution** — resolve differences in units, scales, or encodings for the same attribute across sources.
- **Duplicate elimination** — remove records referring to the same real-world entity that appear more than once after merging.

#### 2.8.3 Data Reduction

**Data reduction** is the process of obtaining a reduced representation of a dataset that is much smaller in volume yet produces the same or nearly the same analytical results. Large datasets make mining computationally expensive and slow. By reducing the data volume — through fewer attributes, fewer records, or a compressed representation — mining algorithms run faster and more efficiently without significant loss of information. Techniques include:
- **Dimensionality reduction** — reduce the number of attributes (e.g., PCA, SVD).
- **Numerosity reduction** — replace the data with a smaller representation such as regression models, histograms, clusters, or samples.
- **Data compression** — encode data in a compressed form.

#### 2.8.4 Data Transformation

**Data transformation** is the process of converting or consolidating data into a format that is more appropriate and effective for data mining. Raw data collected from different sources may be in inconsistent scales, formats, or structures that mining algorithms cannot handle directly. Transformation standardizes the data so that patterns can be discovered more accurately and efficiently. Key techniques:
- **Normalization** — rescale attribute values to a smaller range (min-max, z-score, decimal scaling).
- **Aggregation** — summarize or combine data (e.g., daily sales → monthly sales).
- **Discretization** — convert continuous values into intervals or categories (e.g., age → {young, middle-aged, senior}).
- **Encoding** — convert categorical values into numeric form for algorithms that require numeric input.

#### 2.8.5 Data Sampling

**Sampling** is a data reduction technique that selects a representative subset of the full dataset. It is used when the full dataset is too large to process efficiently.

| Sampling Method | Description |
|---|---|
| **Simple Random Sampling Without Replacement (SRSWOR)** | Each object has equal probability of being chosen; no object is chosen more than once |
| **Simple Random Sampling With Replacement (SRSWR)** | Each object has equal probability; same object can be chosen more than once |
| **Stratified Sampling** | Partition data into strata (groups); sample from each stratum proportionally |
| **Cluster Sampling** | Divide data into clusters; randomly select entire clusters |

A sample is **representative** if it reflects the distribution of the original data. Sampling reduces computational cost while retaining the statistical properties of the full dataset when done correctly.

## 3. OLAP

**OLAP (Online Analytical Processing)** is used to analyze large amounts of historical data. It helps users view data from different dimensions such as product, time, and location. OLAP is mainly used for reporting, business analysis, and decision-making.

### 3.1 OLAP Operations

#### 3.1.1 Slicing

**Slicing** means selecting a subset of a multidimensional data cube by choosing a single value from one or more dimensions.

**Example:** Product A sales — only one product is selected from the product dimension.

#### 3.1.2 Dicing

**Dicing** means selecting data from more than two dimensions of a data cube. It is like applying multiple slices together.

**Example:** Product A sales in 2004 — product and time dimensions are selected.

#### 3.1.3 Drill-Down

**Drill-down** means moving from summarized data to more detailed data. It helps users analyze data in deeper detail.

**Example:** Product A sales in Chicago in 2004 — the data is viewed in more detail using product, location, and time.

#### 3.1.4 Roll-Up

**Roll-up** means summarizing data from detailed level to higher level. It is used to compute summarized data for one or more dimensions.

**Example:** Total product sales — detailed product sales are summarized.

#### 3.1.5 Pivoting

**Pivoting**, also called **rotation**, changes the presentation of data. It rotates the data cube to show the same data from a different view.

**Example:** Changing a report view from product-wise sales to time-wise sales.

### 3.2 OLAP vs OLTP

| S.N. | OLAP | OLTP |
|---|---|---|
| 1 | Stands for Online Analytical Processing | Stands for Online Transaction Processing |
| 2 | Used for data analysis and decision-making | Used for daily transaction processing |
| 3 | Works with historical and summarized data | Works with current and detailed data |
| 4 | Performs complex queries and aggregation operations | Performs simple operations like insert, update, delete, and read |
| 5 | Mainly used by managers, analysts, and data scientists | Mainly used by customers, clerks, and operational staff |
| 6 | Usually uses star schema or snowflake schema | Usually uses normalized database design |
| 7 | Queries are slower — process large amounts of data | Queries are faster — handle small transactions |
| 8 | Example: sales trend analysis | Example: ATM withdrawal or ticket booking |

## 4. Similarity Measures

Similarity measures are used to calculate how similar or different two data objects are. They are useful in data mining, clustering, classification, recommendation systems, and information retrieval.

- Some measures give a **similarity value** — higher value means more similar.
- Some measures give a **distance value** — smaller value means more similar.

Let two data objects be:

$$X = (x_1, x_2, \ldots, x_n)$$

$$Y = (y_1, y_2, \ldots, y_n)$$

### 4.1 Jaccard Similarity

**Jaccard similarity** is used to measure similarity between two sets. It compares the number of common elements with the total number of unique elements in both sets.

$$J(A,B) = \frac{|A \cap B|}{|A \cup B|}$$

Where:
- $A \cap B$ = common elements in both sets
- $A \cup B$ = total unique elements in both sets

The value lies between **0 and 1** — 0 means no similarity, 1 means both sets are exactly the same.

**Example:** If two customers buy some common products, Jaccard similarity can be used to compare their buying patterns.

### 4.2 Cosine Similarity

**Cosine similarity** measures the similarity between two vectors by finding the cosine of the angle between them. It focuses on the direction of the vectors rather than their actual size.

$$\text{Cosine Similarity} = \frac{X \cdot Y}{\|X\| \cdot \|Y\|}$$

Where:
- $X \cdot Y$ = dot product of the two vectors
- $\|X\|$ and $\|Y\|$ = magnitudes of vectors $X$ and $Y$

- Value close to **1** → objects are highly similar.
- Value close to **0** → objects are less similar.

Mostly used in **text mining**, **document similarity**, and **search engines**.

**Example:** Two documents are similar if they contain many common words in similar frequency.

### 4.3 Euclidean Distance

**Euclidean distance** measures the straight-line distance between two points in space. It is one of the most commonly used distance measures.

$$d(X,Y) = \sqrt{\sum_{i=1}^{n}(x_i - y_i)^2}$$

- Smaller distance → more similar.
- Larger distance → less similar.

**Example:** If two points are close to each other on a graph, their Euclidean distance is small.

### 4.4 Simple Matching Coefficient (SMC)

**Simple Matching Coefficient (SMC)** is used to measure similarity between binary data. It considers both types of matches — when both values are 1 and when both values are 0.

$$SMC = \frac{M_{11} + M_{00}}{M_{11} + M_{00} + M_{10} + M_{01}}$$

Where:
- $M_{11}$ = number of attributes where both values are 1
- $M_{00}$ = number of attributes where both values are 0
- $M_{10}$ = number of attributes where first value is 1 and second is 0
- $M_{01}$ = number of attributes where first value is 0 and second is 1

SMC is useful when both presence and absence of attributes are important.

**Example:** It can be used to compare two users based on yes/no answers.

### 4.5 Distance Metric Properties

For a function $d(x, y)$ to qualify as a valid **distance metric**, it must satisfy all four of the following properties:

- **Non-negativity** — $d(x, y) \geq 0$ — distance is always zero or positive.
- **Identity** — $d(x, y) = 0$ if and only if $x = y$ — distance is zero only when both objects are the same.
- **Symmetry** — $d(x, y) = d(y, x)$ — distance from x to y equals distance from y to x.
- **Triangle inequality** — $d(x, z) \leq d(x, y) + d(y, z)$ — the direct distance between two points is never greater than going through a third point.

All Minkowski distance variants (Manhattan, Euclidean, Chebyshev) satisfy these four properties.

### 4.6 Minkowski Distance

**Minkowski distance** is a generalized distance measure. Different values of $p$ produce different distance measures.

$$d(X,Y) = \left(\sum_{i=1}^{n}|x_i - y_i|^p\right)^{\frac{1}{p}}$$

Special cases:
- $p = 1$ → **Manhattan distance**
- $p = 2$ → **Euclidean distance**

A smaller Minkowski distance means higher similarity between objects.

**Example:** Used when a general distance formula is needed that can be adjusted by changing the value of $p$.

### 4.7 Summary

| S.N. | Measure | Type | Used For |
|---|---|---|---|
| 1 | Jaccard Similarity | Similarity | Comparing sets |
| 2 | Cosine Similarity | Similarity | Comparing vectors or documents |
| 3 | Euclidean Distance | Distance | Straight-line distance |
| 4 | SMC | Similarity | Comparing binary data |
| 5 | Minkowski Distance | Distance | General distance calculation |

## 5. Data Normalization Methods

**Data normalization** is a preprocessing technique used to scale data into a smaller and common range. It is useful when attributes have different units or large differences in values. Normalization helps algorithms work better, especially in clustering and classification.

### 5.1 Min-Max Normalization

**Min-max normalization** transforms data into a fixed range, usually between **0 and 1**. It preserves the relationship between original data values.

$$v' = \frac{v - min}{max - min}$$

For a new range $[new\_min, new\_max]$:

$$v' = \frac{v - min}{max - min}(new\_max - new\_min) + new\_min$$

Where $v$ = original value, $v'$ = normalized value.

**Use:** Useful when we know the minimum and maximum values of data.

**Example:** If marks range from 20 to 100, min-max normalization converts all marks into a range from 0 to 1.

### 5.2 Z-Score Normalization

**Z-score normalization** transforms data based on the mean and standard deviation. It shows how far a value is from the mean.

$$v' = \frac{v - \mu}{\sigma}$$

Where $\mu$ = mean, $\sigma$ = standard deviation of the attribute.

- Result = $0$ → value equals the mean
- Result positive → value is above the mean
- Result negative → value is below the mean

**Use:** Useful when data contains outliers or when minimum and maximum values are unknown.

**Example:** If a student's score is higher than the class average, its z-score will be positive.

### 5.3 Decimal Scaling Normalization

**Decimal scaling normalization** normalizes data by moving the decimal point of values. The number of decimal places moved depends on the maximum absolute value in the data.

$$v' = \frac{v}{10^j}$$

Where $j$ is the smallest integer such that $|v'| < 1$.

**Example:** If the maximum value is 986, then $j = 3$, so $986 \rightarrow \frac{986}{10^3} = 0.986$.

**Use:** Simple and useful when values need to be quickly scaled down.

### 5.4 Summary

| S.N. | Method | Description |
|---|---|---|
| 1 | Min-Max Normalization | Scales data into a fixed range, usually 0 to 1 |
| 2 | Z-Score Normalization | Scales data using mean and standard deviation |
| 3 | Decimal Scaling | Scales data by moving the decimal point |
