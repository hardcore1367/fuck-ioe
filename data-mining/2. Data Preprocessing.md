## 1. Introduction 

Real-world data is often incomplete, noisy, and inconsistent, which can lead to incorrect results if used directly. Data preprocessing in data mining is the process of cleaning and preparing raw data so it can be used effectively for analysis and model building.

## 2. Data Types and Attributes
 
An **attribute** (or feature/variable) is a data field representing a characteristic of a data object. 

### Attribute Types

#### Approach 1: By Measurement Scale

-   **Nominal** — values are just different names or labels; they only support equality checks (=, ≠). No ordering or arithmetic is meaningful. Examples: zip codes, employee IDs, eye color, blood type.
-   **Ordinal** — values have a meaningful order or ranking among them, but the actual magnitude of the gap between successive values is not known. Support comparison operators (<, >) but not arithmetic. Examples: hardness of minerals, {good, better, best}, grades, street numbers.
-   **Interval** — differences between values are meaningful because a fixed unit of measurement exists. Support addition and subtraction (+, −) but not multiplication or division, since there is no true zero point. Examples: calendar dates, temperature in °C or °F (0°C does not mean "no temperature").
-   **Ratio** — both differences and ratios are meaningful; a true zero point exists, meaning zero indicates a complete absence of the quantity. Support all arithmetic operations (×, ÷). Examples: temperature in Kelvin, age, mass, length, monetary quantities, counts.

#### Approach 2: By Number of Values

-   **Discrete attribute** — has only a finite or countably infinite set of values, meaning you can list or count all possible values. Often represented as integer variables. Binary attributes are a special case of discrete attributes. Examples: zip codes, counts, the set of words in a document collection.
-   **Continuous attribute** — has real numbers as attribute values and can theoretically take any value within a range. In practice, real values can only be measured and represented using a finite number of digits, so continuous attributes are typically stored as floating-point variables. Examples: temperature, height, weight.

#### Approach 3: By Representation

-   **Character** — values are represented in the form of a character or a set of characters (string). These are qualitative in nature and cannot be used directly in arithmetic operations. Examples: names, labels, codes, descriptions.
-   **Number** — values are represented in numeric form, either as whole numbers (integers) or decimal numbers (real/floating-point). These support mathematical operations and are used in quantitative analysis.

### Symmetric vs Asymmetric Binary Attributes

A **symmetric binary attribute** is one where both states (0 and 1) are equally important and interchangeable. There is no preference on which outcome is coded 0 or 1.

-   Example: _gender_ (male/female) — neither value is more significant.

An **asymmetric binary attribute** is one where the two outcomes are not equally important. By convention, the more important (usually rarer) outcome is coded 1.

-   Example: _HIV test result_ — positive (1) is far more significant than negative (0).


### Dimensionality

**Dimensionality** refers to the number of attributes (features) in a dataset. A **data dimension** is a set of data attributes pertaining to something of interest — things like "customers", "products", "stores", and "time". Higher dimensionality means each data object is described by more attributes.

### Curse of Dimensionality

The **curse of dimensionality** refers to the phenomenon where, as the number of dimensions increases, data becomes increasingly sparse in the space it occupies.

- Definitions of **density and distance between points** become less meaningful — any two points appear roughly equidistant from each other, which breaks distance-based methods like clustering and outlier detection.
- The volume of the feature space grows exponentially, so you need exponentially more data to maintain the same density.
- Many features become irrelevant or redundant, acting as noise and degrading model accuracy.
- Computation and storage costs increase significantly.

### Dimensionality Reduction

**Dimensionality reduction** reduces the number of features while retaining as much important information as possible.

**Purpose:**
- Avoid the curse of dimensionality.
- Reduce the amount of time and memory required by data mining algorithms.
- Allow data to be more easily visualized.
- Help eliminate irrelevant features or reduce noise.

**Techniques:**
- **Principal Component Analysis (PCA)** — transforms original correlated attributes into a smaller set of uncorrelated principal components ordered by variance captured.
- **Singular Value Decomposition (SVD)** — matrix factorization technique closely related to PCA; decomposes a data matrix into singular vectors and values to identify the most important structure.
- **Others** — supervised techniques (use class labels to guide reduction), non-linear techniques such as t-SNE and autoencoders (handle non-linear relationships that PCA cannot).

### PCA — Steps

1. Normalize the data (mean = 0).
2. Compute the **covariance matrix** of the attributes.
3. Find the **eigenvectors and eigenvalues** of the covariance matrix.
4. Sort eigenvectors by eigenvalue descending — higher eigenvalue = more variance captured.
5. Select the top-k eigenvectors as the new axes.
6. Project the original data onto these k eigenvectors → reduced dataset.

PCs are orthogonal (uncorrelated) to each other. PCA is unsupervised — it uses no class label information.

### Data Quality
 
**Data quality** refers to how well data satisfies the requirements of its intended use. The key dimensions are:
 
| Dimension | Meaning |
|---|---|
| **Accuracy** | Data values are correct and free of errors |
| **Completeness** | All required data is present (no missing records/values) |
| **Consistency** | Same data stored across sources agrees with itself |
| **Timeliness** | Data is up-to-date for the intended use |
| **Believability** | Data is trusted by users |
| **Interpretability** | Data is understandable to users |
 
Poor data quality leads to unreliable mining results — "garbage in, garbage out."


### Data Preprocessing Tasks
 
The four major tasks in data preprocessing are:
 
#### 1. Data Cleaning
 
**Data cleaning** fixes or removes incorrect, incomplete, noisy, or inconsistent data. It addresses three main problems:
 
**Missing values** — a value may be absent because it was not recorded, not applicable, or unknown. Ways to handle:
- Ignore the tuple (only suitable if the class label itself is missing).
- Fill in manually (impractical for large datasets).
- Replace with a global constant such as "Unknown" or "N/A".
- Replace with the **attribute mean** (for normally distributed data) or **attribute median** (for skewed data).
- Use the most probable value via inference (e.g., regression or Bayesian methods).
**Noisy data** — random error or variance in measured values caused by faulty instruments, transmission errors, or human error. Handled by binning, regression, or clustering (covered in detail below).
 
**Inconsistencies** — same entity has conflicting values across sources (e.g., two different birthdates for the same customer, or "Age" stored as text in one source and integer in another). Resolved using domain knowledge, metadata, or integrity constraints.
 
#### 2. Data Integration
 
**Data integration** combines data from multiple databases, files, or data cubes into a unified dataset. Key steps and challenges:
- **Schema matching** — identify attributes across sources that refer to the same real-world entity (e.g., "cust_id" vs "customer_no").
- **Redundancy detection** — an attribute derivable from another (e.g., annual_salary = monthly_salary × 12) is redundant and should be removed. Correlation analysis (chi-square for nominal, Pearson's r for numeric) helps detect this.
- **Data conflict resolution** — resolve differences in units, scales, or encodings for the same attribute across sources.
- **Duplicate elimination** — remove records referring to the same real-world entity that appear more than once after merging.
#### 3. Data Reduction
 
**Data reduction** reduces the volume of data while preserving its analytical integrity, making mining faster and more effective. Techniques include:
- **Dimensionality reduction** — reduce the number of attributes (e.g., PCA, SVD).
- **Numerosity reduction** — replace the data with a smaller representation such as regression models, histograms, clusters, or samples.
- **Data compression** — encode data in a compressed form.
#### 4. Data Transformation
 
**Data transformation** converts data into a form suitable for mining. Key techniques:
- **Normalization** — rescale attribute values to a smaller range (min-max, z-score, decimal scaling).
- **Aggregation** — summarize or combine data (e.g., daily sales → monthly sales).
- **Discretization** — convert continuous values into intervals or categories (e.g., age → {young, middle-aged, senior}).
- **Encoding** — convert categorical values into numeric form for algorithms that require numeric input.


# OLAP

## OLAP

**OLAP (Online Analytical Processing)** is used to analyze large amounts of historical data. It helps users view data from different dimensions such as product, time, and location.

OLAP is mainly used for reporting, business analysis, and decision-making.

## OLAP Operations

### 1. Slicing

**Slicing** means selecting a subset of a multidimensional data cube by choosing a single value from one or more dimensions.

**Example:**
Product A sales

Here, only one product is selected from the product dimension.

### 2. Dicing

**Dicing** means selecting data from more than two dimensions of a data cube. It is like applying multiple slices together.

**Example:**
Product A sales in 2004

Here, product and time dimensions are selected.

### 3. Drill-Down

**Drill-down** means moving from summarized data to more detailed data.

It helps users analyze data in deeper detail.

**Example:**
Product A sales in Chicago in 2004

Here, the data is viewed in more detail using product, location, and time.

### 4. Roll-Up

**Roll-up** means summarizing data from detailed level to higher level.

It is used to compute summarized data for one or more dimensions.

**Example:**
Total product sales

Here, detailed product sales are summarized.

### 5. Pivoting

**Pivoting**, also called **rotation**, changes the presentation of data.

It rotates the data cube to show the same data from a different view.

**Example:**
Changing a report view from product-wise sales to time-wise sales.



## OLAP vs OLTP

| S.N. | OLAP                                                                | OLTP                                                                   |
| ---- | ------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| 1    | OLAP stands for **Online Analytical Processing**.                   | OLTP stands for **Online Transaction Processing**.                     |
| 2    | OLAP is used for data analysis and decision-making.                 | OLTP is used for daily transaction processing.                         |
| 3    | OLAP works with historical and summarized data.                     | OLTP works with current and detailed data.                             |
| 4    | OLAP performs complex queries and aggregation operations.           | OLTP performs simple operations like insert, update, delete, and read. |
| 5    | OLAP is mainly used by managers, analysts, and data scientists.     | OLTP is mainly used by customers, clerks, and operational staff.       |
| 6    | OLAP usually uses star schema or snowflake schema.                  | OLTP usually uses normalized database design.                          |
| 7    | OLAP queries are slower because they process large amounts of data. | OLTP queries are faster because they handle small transactions.        |
| 8    | Example of OLAP is sales trend analysis.                            | Example of OLTP is ATM withdrawal or ticket booking.                   |


# Similarity Measures

Similarity measures are used to calculate how similar or different two data objects are. They are useful in data mining, clustering, classification, recommendation systems, and information retrieval.

Some measures give a **similarity value**, where a higher value means more similar.
Some measures give a **distance value**, where a smaller value means more similar.

Let two data objects be:

[
X = (x_1, x_2, ..., x_n)
]

[
Y = (y_1, y_2, ..., y_n)
]

## 1. Jaccard Similarity

**Jaccard similarity** is used to measure similarity between two sets.

It compares the number of common elements with the total number of unique elements in both sets.

[
J(A,B) = \frac{|A \cap B|}{|A \cup B|}
]

Where:

* (A \cap B) means common elements in both sets
* (A \cup B) means total unique elements in both sets

The value of Jaccard similarity lies between **0 and 1**.

* (0) means no similarity
* (1) means both sets are exactly same

**Example:**
If two customers buy some common products, Jaccard similarity can be used to compare their buying pattern.

## 2. Cosine Similarity

**Cosine similarity** measures the similarity between two vectors by finding the cosine of the angle between them.

It focuses on the direction of the vectors rather than their actual size.

[
\text{Cosine Similarity} = \frac{X \cdot Y}{||X|| , ||Y||}
]

Where:

* (X \cdot Y) is the dot product of two vectors
* (||X||) is the magnitude of vector (X)
* (||Y||) is the magnitude of vector (Y)

Cosine similarity is mostly used in **text mining**, **document similarity**, and **search engines**.

If the value is close to **1**, the objects are highly similar.
If the value is close to **0**, the objects are less similar.

**Example:**
Two documents are similar if they contain many common words in similar frequency.

## 3. Euclidean Distance

**Euclidean distance** measures the straight-line distance between two points in space.

It is one of the most commonly used distance measures.

[
d(X,Y) = \sqrt{\sum_{i=1}^{n}(x_i - y_i)^2}
]

Where:

* (x_i) and (y_i) are values of two objects in the (i^{th}) dimension
* (n) is the total number of dimensions

A smaller Euclidean distance means the objects are more similar.
A larger Euclidean distance means the objects are less similar.

**Example:**
If two points are close to each other on a graph, their Euclidean distance is small.

## 4. Simple Matching Coefficient / SMC

**Simple Matching Coefficient (SMC)** is used to measure similarity between binary data.

Binary data contains only two values, usually **0** and **1**.

SMC considers both types of matches:

* When both values are (1)
* When both values are (0)

[
SMC = \frac{M_{11} + M_{00}}{M_{11} + M_{00} + M_{10} + M_{01}}
]

Where:

* (M_{11}) = number of attributes where both values are 1
* (M_{00}) = number of attributes where both values are 0
* (M_{10}) = number of attributes where first value is 1 and second is 0
* (M_{01}) = number of attributes where first value is 0 and second is 1

SMC is useful when both presence and absence of attributes are important.

**Example:**
It can be used to compare two users based on yes/no answers.

## 5. Minkowski Distance

**Minkowski distance** is a generalized distance measure.

It is a flexible formula because different values of (p) produce different distance measures.

[
d(X,Y) = \left(\sum_{i=1}^{n}|x_i - y_i|^p\right)^{\frac{1}{p}}
]

Where:

* (x_i) and (y_i) are values of two objects
* (p) is the order of distance
* (n) is the number of dimensions

Special cases:

* If (p = 1), Minkowski distance becomes **Manhattan distance**
* If (p = 2), Minkowski distance becomes **Euclidean distance**

A smaller Minkowski distance means higher similarity between objects.

**Example:**
It is used when we want a general distance formula that can be adjusted by changing the value of (p).

## Summary

| S.N. | Measure            | Type       | Used For                       |
| ---- | ------------------ | ---------- | ------------------------------ |
| 1    | Jaccard Similarity | Similarity | Comparing sets                 |
| 2    | Cosine Similarity  | Similarity | Comparing vectors or documents |
| 3    | Euclidean Distance | Distance   | Straight-line distance         |
| 4    | SMC                | Similarity | Comparing binary data          |
| 5    | Minkowski Distance | Distance   | General distance calculation   |


# Data Normalization Methods

**Data normalization** is a preprocessing technique used to scale data into a smaller and common range. It is useful when attributes have different units or large differences in values.

Normalization helps algorithms work better, especially in data mining, machine learning, clustering, and classification.

## 1. Min-Max Normalization

**Min-max normalization** transforms data into a fixed range, usually between **0 and 1**.

It preserves the relationship between original data values.

[
v' = \frac{v - min}{max - min}
]

Where:

* (v) = original value
* (v') = normalized value
* (min) = minimum value of the attribute
* (max) = maximum value of the attribute

For a new range ([new_min, new_max]):

[
v' = \frac{v - min}{max - min}(new_max - new_min) + new_min
]

**Example:**
If marks range from 20 to 100, min-max normalization can convert all marks into a range from 0 to 1.

**Use:**
It is useful when we know the minimum and maximum values of data.

## 2. Z-Score Normalization

**Z-score normalization** transforms data based on the mean and standard deviation.

It shows how far a value is from the mean.

[
v' = \frac{v - \mu}{\sigma}
]

Where:

* (v) = original value
* (v') = normalized value
* (\mu) = mean of the attribute
* (\sigma) = standard deviation of the attribute

If the result is:

* (0), the value is equal to the mean
* Positive, the value is above the mean
* Negative, the value is below the mean

**Example:**
If a student's score is higher than the class average, its z-score will be positive.

**Use:**
It is useful when data contains outliers or when the minimum and maximum values are unknown.

## 3. Decimal Scaling Normalization

**Decimal scaling normalization** normalizes data by moving the decimal point of values.

The number of decimal places moved depends on the maximum absolute value in the data.

[
v' = \frac{v}{10^j}
]

Where:

* (v) = original value
* (v') = normalized value
* (j) = smallest integer such that (|v'| < 1)

**Example:**
If the maximum value is 986, then (j = 3).

[
986' = \frac{986}{10^3} = 0.986
]

**Use:**
It is simple and useful when values need to be quickly scaled down.

## Summary

| S.N. | Method                | Description                                    |
| ---- | --------------------- | ---------------------------------------------- |
| 1    | Min-Max Normalization | Scales data into a fixed range, usually 0 to 1 |
| 2    | Z-Score Normalization | Scales data using mean and standard deviation  |
| 3    | Decimal Scaling       | Scales data by moving the decimal point        |
