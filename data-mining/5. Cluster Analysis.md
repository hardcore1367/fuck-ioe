# 5. Cluster Analysis

## 1. Introduction

### 1.1 What is Cluster Analysis
Cluster analysis groups a set of data objects into clusters such that objects within a cluster are similar to each other, and dissimilar to objects in other clusters. It is an unsupervised technique — no predefined class labels.

### 1.2 Clustering vs Classification

| S.N. | Basis | Classification | Clustering |
|---|---|---|---|
| 1 | Learning type | Supervised | Unsupervised |
| 2 | Class labels | Known in advance (training data labeled) | Not known; discovered from data |
| 3 | Goal | Predict class of new instance | Group similar objects together |
| 4 | Output | Class label per instance | Set of clusters |

### 1.3 Approaches to Clustering
- **Hierarchical methods:** Build a tree of clusters (agglomerative or divisive).
- **Non-hierarchical (partitional) methods:** Specify number of clusters in advance and partition data directly (e.g., K-means, DBSCAN).

## 2. K-Means Clustering

### 2.1 Algorithm
1. Choose K initial cluster centers (centroids), often chosen far apart.
2. Assign each object to its nearest centroid (by distance, e.g. Euclidean).
3. Recompute centroid of each cluster as mean of its members.
4. Repeat steps 2–3 until centroids stabilize (no change / convergence).

### 2.2 Limitations / Problems
- Number of clusters (K) must be specified in advance.
- Sensitive to initial centroid selection — different runs can give different results.
- Sensitive to outliers, since mean is affected by extreme values.
- Works well only for spherical/convex-shaped clusters of similar size and density.
- Can converge to a local optimum, not the global best solution.

### 2.3 Advantages and Disadvantages

| S.N. | Advantages | Disadvantages |
|---|---|---|
| 1 | Simple and easy to implement | Must specify K in advance |
| 2 | Efficient for large datasets (linear time) | Sensitive to initial centroid choice |
| 3 | Converges quickly in practice | Sensitive to outliers and noise |
| 4 | Works well for compact, well-separated clusters | Fails on non-convex/arbitrary-shaped clusters |

### 2.4 Bisecting K-Means
- A variant that avoids fixing K upfront by repeated splitting.
- Steps:
  1. Start with all points in a single cluster.
  2. Split the largest/highest-SSE cluster into two using basic K-means (K=2).
  3. Repeat splitting until the desired number of clusters is obtained.
- Produces more stable results than plain K-means since it reduces sensitivity to initial centroid choice.

## 3. Hierarchical Clustering

### 3.1 Agglomerative vs Divisive

| S.N. | Basis | Agglomerative | Divisive |
|---|---|---|---|
| 1 | Direction | Bottom-up | Top-down |
| 2 | Start | Each object is its own cluster | All objects in one cluster |
| 3 | Process | Merge closest pair of clusters repeatedly until one cluster remains | Split cluster repeatedly until each object is its own cluster |
| 4 | Usage | More commonly used | Less commonly used |

### 3.2 Agglomerative Algorithm (Johnson's Method)
1. Assign each item to its own cluster (N items → N clusters).
2. Find the closest pair of clusters and merge into one cluster.
3. Recompute distance between the new cluster and remaining clusters.
4. Repeat steps 2–3 until all items form a single cluster.

### 3.3 Linkage Methods (how distance between clusters is defined)

| S.N. | Method | Distance definition | Note |
|---|---|---|---|
| 1 | Single linkage (nearest neighbor) | Shortest distance between any two members of the two clusters | Can cause "chaining" (long straggly clusters) |
| 2 | Complete linkage (furthest neighbor) | Longest distance between any two members | Produces compact, similar-sized clusters; sensitive to outliers |
| 3 | Average linkage | Average distance over all pairs of members | Fairly robust |
| 4 | Centroid method | Distance between cluster centroids (means) | Fairly robust |
| 5 | Ward's method | Merge that gives minimum increase in total within-cluster sum of squares | Produces near equal-sized clusters; sensitive to outliers |

### 3.4 Dendrogram
A dendrogram is a tree diagram that records the sequence of merges (or splits) in hierarchical clustering, showing the distance/similarity at which each merge occurred. Cutting the dendrogram at a chosen height gives a specific number of clusters.

### 3.5 Advantages and Disadvantages

| S.N. | Advantages | Disadvantages |
|---|---|---|
| 1 | No need to specify number of clusters in advance | Once a merge/split is done it cannot be undone (no backtracking) |
| 2 | Dendrogram gives useful visualization of nested clusters | Computationally expensive for large datasets: $O(n^2 \log n)$ or worse |
| 3 | Can use any distance/similarity measure | Sensitive to noise and outliers (esp. single/complete linkage) |

## 4. DBSCAN Clustering

### 4.1 Core Idea
DBSCAN (Density-Based Spatial Clustering of Applications with Noise) groups together points that are closely packed (dense regions), marking points in low-density regions as noise/outliers. Needs two parameters: **epsilon (ε)** — neighborhood radius, and **minPoints** — minimum number of points required to form a dense region.

### 4.2 Point Types

| S.N. | Point type | Definition |
|---|---|---|
| 1 | Core point | Has at least minPoints points (including itself) within distance ε |
| 2 | Border point | Has fewer than minPoints within ε, but lies within ε of a core point |
| 3 | Noise point | Neither a core point nor a border point; not part of any cluster |

### 4.3 Density Reachability and Connectivity
- **Directly density-reachable:** Point q is directly density-reachable from p if q is within ε of p and p is a core point.
- **Density-reachable:** q is density-reachable from p if there is a chain of points p1, p2, …, pn (p1 = p, pn = q) where each pi+1 is directly density-reachable from pi.
- **Density-connected:** Points p and q are density-connected if there exists a point o such that both p and q are density-reachable from o.

### 4.4 Algorithm
1. Pick an arbitrary unvisited point.
2. If it is a core point (≥ minPoints within ε), start a new cluster and expand it by recursively adding all density-reachable points.
3. If it is not a core point and not reachable from any cluster, mark it as noise.
4. Repeat until all points are visited.

### 4.5 Advantages and Disadvantages

| S.N. | Advantages | Disadvantages |
|---|---|---|
| 1 | No need to specify number of clusters in advance | Choosing good ε and minPts is difficult |
| 2 | Can find arbitrarily shaped clusters | Struggles when clusters have widely varying densities |
| 3 | Naturally identifies outliers/noise | Not efficient for very high-dimensional data |

## 5. Cluster Evaluation and Issues

### 5.1 Why Evaluate
Cluster validation checks whether the clusters found are meaningful and not an artifact of the algorithm, and compares different clustering results.

### 5.2 Sum of Squared Errors (SSE)
SSE measures compactness of clusters — the sum of squared distances between each point and its cluster's centroid:

$$SSE = \sum_{i=1}^{K} \sum_{x \in C_i} dist(x, centroid_i)^2$$

- Lower SSE means points are closer to their centroids (more compact clusters).
- Used to compare different K values or different runs of K-means (e.g., "elbow method" — plot SSE vs K, choose K at the bend).

### 5.3 Evaluation Measures
- **Cohesion:** How closely related objects in a cluster are (measured via SSE).
- **Separation:** How distinct/well-separated a cluster is from other clusters.
- **Silhouette coefficient:** Combines cohesion and separation into a single score per point.

### 5.4 Issues in Cluster Analysis
- **Scalability:** Algorithms must handle very large datasets efficiently; some methods (e.g., basic hierarchical clustering) do not scale well due to high time/space complexity.
- **Comparison of methods:** No single clustering algorithm works best for all data shapes/sizes/densities — choice depends on data characteristics (K-means for spherical clusters, DBSCAN for arbitrary shape and noise handling, hierarchical for dendrogram-based analysis).
- **Determining number of clusters:** Choosing the right K or ε/minPoints is often non-trivial and affects result quality.
