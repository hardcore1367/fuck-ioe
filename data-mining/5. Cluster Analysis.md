
# Chapter 5: Cluster Analysis

## 1. Introduction

### 1.1 What is Cluster Analysis
Cluster analysis is the process of grouping a set of data objects into clusters, in such a way that objects within the same cluster are highly similar to one another, while objects in different clusters are as dissimilar as possible. It is an **unsupervised** technique, meaning the data has no predefined class labels — the algorithm itself discovers the grouping structure hidden in the data based on similarity/distance between objects.

![Cluster Analysis](https://github.com/user-attachments/assets/09c8a6cb-a4c1-46f8-bc01-db26c20acf24)

### 1.2 Clustering vs Classification
Classification and clustering are often confused because both group data, but they work very differently:

| S.N. | Basis | Classification | Clustering |
|---|---|---|---|
| 1 | Learning type | Supervised — model learns from labeled training data | Unsupervised — no labels are given |
| 2 | Class labels | Known in advance for training data | Not known; discovered from the data itself |
| 3 | Goal | Predict the class of a new/unseen instance | Group similar objects together into meaningful clusters |
| 4 | Output | A class label assigned to each instance | A set of clusters, with no predefined meaning attached |

### 1.3 Approaches to Clustering
Broadly, clustering methods fall into two categories:
- **Hierarchical methods:** These build a nested tree (hierarchy) of clusters, either by merging smaller clusters into bigger ones (agglomerative) or by splitting one big cluster into smaller ones (divisive).
- **Non-hierarchical (partitional) methods:** These require the number of clusters to be decided beforehand, and directly partition the data into that many groups. K-means and DBSCAN both fall in this category.


## 2. K-Means Clustering

### 2.1 Algorithm
K-means is a partitional method that divides data into K clusters, where K is chosen by the user. The steps are:
1. Choose K initial cluster centers (centroids) — usually picked so that they are reasonably far apart from each other.
2. Assign every data point to the nearest centroid, based on a distance measure (commonly Euclidean distance). This forms K initial clusters.
3. Recalculate the centroid of each cluster by taking the mean of all points currently assigned to it.
4. Repeat steps 2 and 3 — reassigning points and recomputing centroids — until the centroids no longer change significantly (i.e., the algorithm has converged).

![K-Means Clustering](https://github.com/user-attachments/assets/36a6fad8-2188-4223-8e69-d4ac927d4a74)


### 2.2 Limitations / Problems
- The number of clusters K has to be specified beforehand, but the right value of K is often not known in advance.
- The final result depends heavily on the initial choice of centroids — different starting points can lead to different final clusters.
- Since the centroid is a mean value, K-means is sensitive to outliers, which can pull the centroid away from the true center of the cluster.
- It assumes clusters are roughly spherical/convex and of similar size and density, so it performs poorly on irregularly shaped or overlapping clusters.
- The algorithm can get stuck in a local optimum instead of finding the best possible clustering.

### 2.3 Advantages and Disadvantages
**Advantages:**
- Simple to understand and implement.
- Computationally efficient, so it scales well to large datasets.
- Usually converges in relatively few iterations.
- Works well when clusters are compact and well separated.

**Disadvantages:**
- Requires K to be known/guessed beforehand.
- Sensitive to the initial placement of centroids.
- Easily affected by outliers and noisy data.
- Cannot handle non-convex or arbitrarily shaped clusters well.

### 2.4 Bisecting K-Means
Bisecting K-means is a variant designed to reduce K-means' sensitivity to initial centroid choice and to avoid deciding K all at once. Instead, it builds up the clusters gradually:
1. Start with all the data points in a single cluster.
2. Pick a cluster to split (usually the one with the largest size or highest SSE) and apply basic K-means with K=2 on it to split it into two smaller clusters.
3. Repeat this splitting process on the resulting clusters until the desired total number of clusters is reached.

Because each split is a small, simpler 2-means problem, this approach tends to give more consistent and stable results than running plain K-means directly with the full K.


## 3. Hierarchical Clustering

### 3.1 Agglomerative vs Divisive
Hierarchical clustering builds a tree-like structure of nested clusters, and it can proceed in two opposite directions:

![Hierarchical Clustering](https://github.com/user-attachments/assets/0232d61c-e234-4d2c-9fe6-7bd63eeeb276)


| S.N. | Basis | Agglomerative | Divisive |
|---|---|---|---|
| 1 | Direction | Bottom-up | Top-down |
| 2 | Starting point | Every object starts in its own separate cluster | All objects start together in one big cluster |
| 3 | Process | The two closest clusters are merged repeatedly until only one cluster remains | The cluster is split repeatedly until every object is in its own cluster |
| 4 | Usage in practice | Much more commonly used | Rarely used, since splitting decisions are computationally harder |

### 3.2 Agglomerative Algorithm (Johnson's Method)
The general procedure for agglomerative clustering is:
1. Start by treating each of the N data items as its own cluster, so there are initially N clusters.
2. Find the pair of clusters that are closest (most similar) to each other, and merge them into a single cluster.
3. Recalculate the distance between this newly formed cluster and all the remaining clusters.
4. Keep repeating steps 2 and 3, merging one pair at a time, until all items belong to a single cluster.

### 3.3 Linkage Methods
A key decision in hierarchical clustering is how exactly to measure the "distance" between two clusters, since each cluster contains multiple points. Different linkage methods answer this differently:

| S.N. | Method | How distance is defined | Remark |
|---|---|---|---|
| 1 | Single linkage (nearest neighbor) | Shortest distance between any point in one cluster and any point in the other | Can cause "chaining", producing long straggly clusters |
| 2 | Complete linkage (furthest neighbor) | Longest distance between any point in one cluster and any point in the other | Produces compact, similarly sized clusters, but is sensitive to outliers |
| 3 | Average linkage | Average of all pairwise distances between points of the two clusters | Considered fairly robust |
| 4 | Centroid method | Distance between the mean (centroid) of one cluster and the mean of the other | Also fairly robust |
| 5 | Ward's method | Chooses the merge that causes the smallest increase in total within-cluster variance | Tends to produce clusters of roughly equal size, but is sensitive to outliers |

### 3.4 Dendrogram
A dendrogram is a tree-shaped diagram used to visualize the sequence of merges (or splits) performed during hierarchical clustering. The height at which two branches join represents the distance (or dissimilarity) at which those two clusters were merged. By cutting the dendrogram horizontally at a chosen height, we can obtain any desired number of clusters without re-running the algorithm.

### 3.5 Advantages and Disadvantages
**Advantages:**
- Does not require the number of clusters to be specified in advance.
- The dendrogram gives a clear, intuitive visualization of how clusters are nested within each other.
- Can work with any chosen distance or similarity measure.

**Disadvantages:**
- Once a merge (or split) is made, it cannot be undone later, even if it turns out to be a poor decision.
- Computationally expensive for large datasets, since distances between many cluster pairs must be repeatedly recalculated.
- Sensitive to noise and outliers, especially with single and complete linkage.


## 4. DBSCAN Clustering

### 4.1 Core Idea
DBSCAN (Density-Based Spatial Clustering of Applications with Noise) is based on the idea that a cluster is a region where data points are densely packed together, separated by regions of low density. Instead of relying on distance to a centroid, it looks at how many neighbors each point has nearby. It requires two parameters: **epsilon (ε)**, the radius that defines a point's neighborhood, and **minPoints**, the minimum number of points required within that radius for the region to be considered dense.

![DBSCAN Clustering](https://github.com/user-attachments/assets/9ec32ebb-d387-49f9-afe6-de1c8468ae30)


### 4.2 Point Types
Based on the density around each point, DBSCAN classifies every point into one of three types:

| S.N. | Point type | Definition |
|---|---|---|
| 1 | Core point | A point that has at least minPoints points (including itself) within distance ε — it lies inside a dense region |
| 2 | Border point | A point that does not have enough neighbors to be a core point itself, but lies within ε of some core point |
| 3 | Noise point | A point that is neither a core point nor a border point — it does not belong to any cluster |

### 4.3 Density Reachability and Connectivity
These concepts describe how clusters are formed from individual core and border points:
- **Directly density-reachable:** A point q is directly density-reachable from p if q lies within ε of p, and p itself is a core point.
- **Density-reachable:** A point q is density-reachable from p if there is a chain of points connecting them, where each point in the chain is directly density-reachable from the previous one.
- **Density-connected:** Two points p and q are density-connected if there is some other point from which both p and q are density-reachable. This connectivity is what ties multiple points together into a single cluster.

### 4.4 Algorithm
1. Pick an arbitrary point that has not yet been visited.
2. If this point is a core point, start a new cluster from it and expand the cluster by adding all points that are density-reachable from it.
3. If the point is not a core point and is not reachable from any existing cluster, temporarily label it as noise.
4. Repeat this process for every unvisited point until all points have been processed.

### 4.5 Advantages and Disadvantages
**Advantages:**
- Does not require the number of clusters to be specified beforehand.
- Can discover clusters of arbitrary shape, not just spherical ones.
- Automatically identifies outliers/noise points, which is useful in real-world noisy data.

**Disadvantages:**
- Choosing appropriate values for ε and minPoints can be difficult and greatly affects the result.
- Struggles when clusters in the same dataset have very different densities.
- Becomes less effective and less efficient as the number of dimensions (attributes) grows very high.


## 5. Cluster Evaluation and Issues

### 5.1 Why Evaluate
After clustering is performed, it is important to validate whether the resulting clusters are actually meaningful and reflect real structure in the data, rather than being an artifact of the particular algorithm or parameters used. Evaluation also allows different clustering results (e.g., different values of K, or different algorithms) to be compared against each other.

### 5.2 Sum of Squared Errors (SSE)
SSE is one of the most common measures used to evaluate the compactness of clusters, particularly for K-means. It sums the squared distance of every point from the centroid of the cluster it belongs to:

$$SSE = \sum_{i=1}^{K} \sum_{x \in C_i} dist(x, centroid_i)^2$$

A lower SSE indicates that points are, on average, closer to their respective centroids, meaning the clusters are more compact. SSE is commonly used to compare clustering results for different values of K — for example, in the "elbow method", SSE is plotted against K, and the value of K at which the curve bends sharply (the "elbow") is chosen as a good balance between accuracy and simplicity.

### 5.3 Evaluation Measures
Besides SSE, a few other concepts are commonly used to judge cluster quality:
- **Cohesion:** Measures how tightly related the objects within a single cluster are to one another (closely tied to SSE — the smaller the internal distances, the higher the cohesion).
- **Separation:** Measures how distinct and well-separated a cluster is from other clusters.
- **Silhouette coefficient:** A combined measure that takes both cohesion and separation into account, giving a single score (usually between -1 and 1) for how well each point fits within its assigned cluster compared to other clusters.

### 5.4 Issues in Cluster Analysis
- **Scalability:** Clustering algorithms need to handle very large datasets efficiently; some methods, such as basic hierarchical clustering, become impractical on large data due to high time and space complexity.
- **Comparison of methods:** No single clustering algorithm performs best on every kind of data — the right choice depends on the shape, size, and density of the data (e.g., K-means suits compact spherical clusters, DBSCAN suits arbitrarily shaped clusters with noise, hierarchical clustering suits cases where a nested/dendrogram view is useful).
- **Determining the number of clusters:** Deciding the right value of K (for K-means) or the right ε and minPoints (for DBSCAN) is often not straightforward, and a poor choice can significantly affect the quality of the final clusters.
