## Multiplexed Display

A **multiplexed display** is a technique to drive multiple 7-segment displays using fewer I/O pins by turning on each display one at a time so rapidly that the human eye perceives all digits as continuously lit — this is called **persistence of vision**.

Instead of dedicating separate segment lines to each digit, all displays share the same **segment bus** (lines a–g), and each digit is selected individually via its **common pin**. The controller cycles through each digit, sends the correct segment data, then moves to the next — completing the full cycle faster than ~50 Hz so no flicker is visible.

## Key Points

- **Pin saving:** A 4-digit static display needs 32 I/O lines; multiplexing reduces this to 8 segment + 4 common = **12 lines**.
- **Duty cycle:** Each digit is ON for 1/N of the total cycle (N = number of digits). For 4 digits, duty cycle = 25% per digit.
- **Brightness compensation:** Since each digit is ON only part of the time, drive current is increased during the ON period to maintain adequate brightness.
- **Control overhead:** A microcontroller or dedicated IC must continuously refresh the display; this is a software/hardware tradeoff for the pin savings gained.
- **Applications:** Digital clocks, calculators, scoreboards, and embedded system readouts.

## Frequency Counter

A **frequency counter** is a digital instrument that measures how many times a signal repeats per second (Hz). It works by counting the pulses of an input signal over a fixed time window called the **gate time**, controlled by a precise crystal oscillator.

## Key Points

- **Counting principle:** Counter counts input pulses for the gate period (e.g., 1 sec). If 5000 pulses are counted in 1 sec → frequency = **5000 Hz**.
- **Signal conditioning:** Input signal passes through an **amplifier** and **Schmitt trigger** to convert any waveform into clean countable digital pulses.
- **Gate time:** Longer gate = better resolution but slower update. Shorter gate = faster update but less accurate.
- **Prescaler:** Divides high-frequency input (÷10, ÷100) so the counter can handle MHz/GHz range signals.
- **Display:** Count is latched at end of gate period and shown on a 7-segment display, then counter resets for next cycle.
- **Applications:** Lab instruments, RF measurement, clock signal verification, signal generators.

## Digital Clock

A **digital clock** is an electronic timekeeping circuit that displays time in hours, minutes, and seconds using 7-segment displays. It uses a **crystal oscillator** as its time base to generate accurate, stable pulses which are then counted and divided down to produce 1 Hz ticks for seconds counting.

## Key Points

- **Crystal oscillator** generates a high-frequency signal (commonly 32.768 kHz) which is divided down by a **frequency divider** circuit to produce exactly 1 pulse per second.
- **Counters** chain together to count seconds (0–59), minutes (0–59), and hours (0–23 or 1–12), each resetting and carrying over to the next stage when their limit is reached.
- **BCD counters** are commonly used since they count in **Binary Coded Decimal**, making it easy to drive 7-segment displays directly via a BCD-to-7-segment decoder.
- **Multiplexed display** is used to show all digits (HH:MM:SS) using fewer I/O lines by rapidly switching between digits faster than the eye can detect.
- **Reset and carry logic** ensures seconds roll over to 0 and trigger minutes to increment, and similarly minutes trigger hours — forming a complete cascaded counting chain.
- **Applications:** Wall clocks, alarm clocks, embedded real-time systems, microcontroller-based timers.

  ## Time Measurement

**Time measurement** in digital logic refers to measuring the duration or time interval between two events using digital counting techniques. A precise **clock oscillator** generates known-frequency pulses, and the number of pulses counted between a START and STOP event directly gives the elapsed time.

## Key Points

- **Basic principle:** A counter starts counting clock pulses when a START signal arrives and stops when a STOP signal arrives. Elapsed time = **pulse count × clock period**.
- **Crystal oscillator** provides the time base — a 1 MHz clock has a period of 1 µs, so each counted pulse = 1 µs of elapsed time.
- **Resolution** depends on clock frequency — higher clock frequency means smaller time intervals can be detected (better resolution).
- **Gate control logic** uses flip-flops to open and close the counting window precisely on the START and STOP edges.
- **BCD counters and decoders** convert the count into a readable format on 7-segment displays, similar to a digital clock or stopwatch.
- **Overflow handling:** For long time intervals, cascaded counters (seconds, minutes, hours) are used to extend the measurable range.
- **Applications:** Digital stopwatches, pulse width measurement, frequency counters (gate time), event timers, and speed measurement systems.
