# Relational algebra cheatsheet

---

## Operators at a glance

| Symbol | Name | Reads as |
|---|---|---|
| σ | Selection | "where" |
| π | Projection | "select columns" |
| ⋈ | Natural join | "join on common attributes" |
| ⋈θ | Theta join | "join on condition θ" |
| ρ | Rename | "call it" |
| ℊ | Group by aggregate | "group by … compute …" |
| ∪ | Union | "all rows from both" |
| − | Difference | "in first but not second" |
| ∩ | Intersection | "in both" |
| × | Cartesian product | "every combination" |
| ÷ | Division | "for every" |
| ← | Assignment | "store result as" |

---

## σ — Selection
Filters **rows**. Condition goes in subscript.

```
σ(Salary > 50000)(Employee)
σ(DeptID = 3 ∧ Salary > 40000)(Employee)   -- AND
σ(City = 'KTM' ∨ City = 'PKR')(Branch)     -- OR
σ(NOT DeptID = 2)(Employee)                 -- NOT
```
SQL → `WHERE`

---

## π — Projection
Keeps only specified **columns**. Removes duplicate rows automatically.

```
π(Name, Salary)(Employee)
π(Name, Salary)(σ(Salary > 50000)(Employee))   -- compose with selection
```
SQL → `SELECT col1, col2`

---

## ⋈ — Natural join
Joins on **all attributes with the same name**, keeps one copy.

```
Employee ⋈ Department
Employee ⋈ WorksOn ⋈ Project          -- chain joins
```
SQL → `NATURAL JOIN`

---

## ⋈θ — Theta join (condition join)
Joins on an explicit condition θ.

```
Employee ⋈(Employee.DeptID = Department.DeptID) Department

-- inequality join
Loan ⋈(Loan.Amount > Account.Balance) Account
```
When θ uses only `=` it is called an **equi-join**.
SQL → `JOIN … ON …`

---

## ρ — Rename
Used to **alias** a relation or its attributes — essential for self-joins.

```
ρ(Emp1, Employee)                        -- rename relation only
ρ(Emp1(ID, EName, Sal), Employee)        -- rename relation + attributes

-- self-join: find employees who earn more than their manager
ρ(E1, Employee) ⋈(E1.ManagerID = E2.EmpID ∧ E1.Salary > E2.Salary) ρ(E2, Employee)
```
SQL → `AS` alias

---

## ℊ — Group by aggregate
Format: `ℊ grouping-attrs ; agg-function(attr) (Relation)`

```
ℊ DeptID ; COUNT(EmpID)(Employee)
ℊ DeptID ; AVG(Salary), MAX(Salary)(Employee)
ℊ ; COUNT(EmpID)(Employee)             -- no grouping = whole table
```
SQL → `GROUP BY … ` + `COUNT / SUM / AVG / MAX / MIN`

To filter groups, apply σ after:
```
σ(COUNT > 3)(ℊ DeptID ; COUNT(EmpID)(Employee))
```
SQL equivalent of `HAVING`

---

## Set operations — ∪ − ∩
Both relations must be **union-compatible** (same number of attributes, compatible types).

```
π(Name)(Customer) ∪ π(Name)(Employee)     -- all names in either
π(Name)(Customer) − π(Name)(Employee)     -- customers who are not employees
π(Name)(Customer) ∩ π(Name)(Employee)     -- people who are both
```

| Symbol | SQL |
|---|---|
| ∪ | `UNION` |
| − | `EXCEPT` |
| ∩ | `INTERSECT` |

---

## × — Cartesian product
Every row of R paired with every row of S. Rarely used alone; always follow with σ.

```
Employee × Department
σ(Employee.DeptID = Department.DeptID)(Employee × Department)
-- ↑ this is exactly what a theta join does
```

---

## ÷ — Division
Answers **"for all"** queries. R ÷ S returns tuples in R that are paired with **every** tuple in S.

Use when you see: *"find X who did all Y"*, *"customers who bought every product"*

```
-- Find employees who worked on ALL projects
π(EmpID, ProjectID)(WorksOn) ÷ π(ProjectID)(Project)
```

**Manual derivation** (if ÷ is not allowed):
```
-- Let
  T1 ← π(EmpID)(WorksOn)
  T2 ← π(EmpID, ProjectID)(T1 × π(ProjectID)(Project))
  T3 ← T2 − π(EmpID, ProjectID)(WorksOn)
  Result ← T1 − π(EmpID)(T3)
```
SQL → double `NOT EXISTS` (see SQL cheatsheet)

---

## ← — Assignment
Stores an intermediate result to keep expressions readable.

```
Temp ← σ(Salary > 60000)(Employee)
Result ← π(Name, DeptID)(Temp)
```
Write multi-step solutions this way in exam answers.

---

## Operator precedence (high → low)

1. σ, π, ρ (unary — applied first)
2. × , ⋈ (cartesian / joins)
3. ∩
4. ∪ , −

Use parentheses whenever in doubt.

---

## Composing expressions — worked examples

**Find names of employees in the 'HR' department**
```
π(Name)(σ(DeptName = 'HR')(Employee ⋈ Department))
```

**Find names of employees working on project 'Apollo'**
```
π(Name)(
  σ(ProjectName = 'Apollo')(
    Employee ⋈ WorksOn ⋈ Project
  )
)
```

**Find employees who work on NO project**
```
π(EmpID)(Employee) − π(EmpID)(WorksOn)
```

**Find departments with more than 3 employees**
```
σ(EmpCount > 3)(ℊ DeptID ; COUNT(EmpID) AS EmpCount(Employee))
```

**Find customers who have taken loans in ALL branches**
```
π(CustomerID, BranchID)(Loan) ÷ π(BranchID)(Branch)
```

---

## Quick reminders

- σ and π can be **composed**: apply σ first to reduce rows, then π to reduce columns
- ÷ is derived — it can always be rewritten using π, ×, and −
- ρ is **mandatory** in self-joins, otherwise attribute names are ambiguous
- Natural join can give **wrong results** if two unrelated attributes happen to share a name — prefer theta join when in doubt
- Set operations require **union compatibility** — check before using ∪ − ∩
