# SQL cheatsheet

---

## DDL — Data definition language

### CREATE TABLE
```sql
CREATE TABLE Employee (
  EmpID   INT          PRIMARY KEY,
  Name    VARCHAR(50)  NOT NULL,
  Email   VARCHAR(100) UNIQUE,
  Salary  DECIMAL(10,2),
  DeptID  INT,
  FOREIGN KEY (DeptID) REFERENCES Department(DeptID)
);
```

### CREATE VIEW
```sql
CREATE VIEW EmpDept AS
SELECT e.Name, d.DeptName
FROM Employee e
JOIN Department d ON e.DeptID = d.DeptID;
```

## DML — Data manipulation language

### SELECT + WHERE + ORDER BY
```sql
SELECT Name, Salary
FROM Employee
WHERE Salary > 50000
ORDER BY Salary DESC;
```

### INNER JOIN
```sql
SELECT e.Name, d.DeptName
FROM Employee e
INNER JOIN Department d ON e.DeptID = d.DeptID;
```

### NATURAL JOIN
```sql
SELECT * FROM Employee NATURAL JOIN Department;
-- joins on all columns sharing the same name
```

### GROUP BY + HAVING
```sql
SELECT DeptID, COUNT(*) AS Total, AVG(Salary) AS AvgSal
FROM Employee
GROUP BY DeptID
HAVING COUNT(*) > 3;
-- WHERE filters rows; HAVING filters groups
```

### LIKE — pattern matching
| Pattern | Matches |
|---|---|
| `'A%'` | starts with A |
| `'%a'` | ends with a |
| `'%a%'` | contains a anywhere |
| `'_b%'` | any single char, then b, then anything |

```sql
SELECT * FROM Employee WHERE Name LIKE 'A%';
```

### Subqueries
```sql
-- IN / NOT IN
SELECT Name FROM Employee
WHERE DeptID IN (SELECT DeptID FROM Department WHERE City = 'Kathmandu');

-- EXISTS
SELECT Name FROM Employee e
WHERE EXISTS (
  SELECT 1 FROM Project p WHERE p.ManagerID = e.EmpID
);

-- Scalar subquery in WHERE
SELECT Name FROM Employee
WHERE Salary > (SELECT AVG(Salary) FROM Employee);

-- Subquery in FROM (derived table)
SELECT d.DeptName, sub.AvgSal
FROM (SELECT DeptID, AVG(Salary) AS AvgSal FROM Employee GROUP BY DeptID) sub
JOIN Department d ON d.DeptID = sub.DeptID;
```

### INSERT
```sql
INSERT INTO Employee (EmpID, Name, Salary, DeptID)
VALUES (101, 'Rajan', 45000, 3);
```

### UPDATE + CASE WHEN
```sql
UPDATE Employee
SET Salary = CASE
  WHEN Salary < 30000 THEN Salary * 1.20
  WHEN Salary < 60000 THEN Salary * 1.10
  ELSE Salary * 1.05
END
WHERE DeptID = 2;
```

### DELETE
```sql
DELETE FROM Employee
WHERE EmpID = 101;
```

## Aggregate functions

| Function | Use |
|---|---|
| `COUNT(*)` | total rows |
| `COUNT(col)` | non-null values |
| `SUM(col)` | total |
| `AVG(col)` | average |
| `MAX(col)` | highest value |
| `MIN(col)` | lowest value |

```sql
SELECT COUNT(*), SUM(Salary), AVG(Salary), MAX(Salary), MIN(Salary)
FROM Employee
WHERE DeptID = 1;
```

## Relational algebra

| Symbol | Name | SQL equivalent |
|---|---|---|
| σ | Selection | `WHERE` |
| π | Projection | `SELECT col1, col2` |
| ⋈ | Natural / theta join | `NATURAL JOIN` / `JOIN ON` |
| ρ | Rename | `AS` alias (needed for self-join) |
| ℊ | Group by aggregate | `GROUP BY` + aggregate |
| ∪ | Union | `UNION` |
| − | Difference | `EXCEPT` |
| ∩ | Intersection | `INTERSECT` |
| ÷ | Division | "for all" — use `NOT IN` + `NOT EXISTS` |
| ← | Assignment | used in update/delete expressions |

### Examples
```
-- Selection + projection
π(Name, Salary) σ(Salary > 50000) (Employee)

-- Theta join
Employee ⋈(Employee.DeptID = Department.DeptID) Department

-- Self-join using rename
ρ(E1, Employee) ⋈(E1.ManagerID = E2.EmpID) ρ(E2, Employee)

-- Division (find employees who worked on ALL projects)
π(EmpID, ProjectID)(WorksOn) ÷ π(ProjectID)(Project)
```

### Division in SQL (NOT IN + NOT EXISTS pattern)
```sql
-- "Find employees assigned to ALL projects"
SELECT DISTINCT EmpID FROM WorksOn w1
WHERE NOT EXISTS (
  SELECT ProjectID FROM Project p
  WHERE NOT EXISTS (
    SELECT * FROM WorksOn w2
    WHERE w2.EmpID = w1.EmpID AND w2.ProjectID = p.ProjectID
  )
);
```

## Quick reminders

- `WHERE` runs **before** grouping → use it to filter rows
- `HAVING` runs **after** grouping → use it to filter groups
- `DISTINCT` removes duplicate rows: `SELECT DISTINCT DeptID FROM Employee`
- Aliases don't work in the same `WHERE`/`HAVING` clause they're defined in
- `NULL` comparisons always use `IS NULL` / `IS NOT NULL`, never `= NULL`
