# Machine Learning

Learning is an ability of an agent to improve it's behavious based on experience.

Machine Learning is a subset of AI that is mainly concerned with the development of algorithms which allow a computer to learn from the data and past experiences on their own.

## 1. Learning Types

### 1. Supervised Learning

Supervised learning is a type of machine learning where the model is trained on **labeled data**, meaning each input in the training set already has a known correct output. The algorithm learns by studying these input-output pairs, finding patterns, and building a mapping function that connects inputs to their correct outputs. Once training is complete, the model uses this learned mapping to predict outputs for new, unseen inputs. It is called "supervised" because the correct answers guide the learning process, just like a teacher supervising a student.

**Types**
- **Classification** — output is a category (e.g., spam or not spam, disease or no disease).
- **Regression** — output is a continuous number (e.g., predicting house price, temperature).

**Key characteristics:**
- Requires a large amount of labeled training data.
- Feedback is direct — the model always knows if it was right or wrong during training.
- Easier to evaluate because expected outputs are known.

**Example:** Training a model with thousands of emails already marked "spam" or "not spam" so it can classify future emails on its own.

### 3. Unsupervised Learning

Unsupervised learning is a type of machine learning where the model is trained on **unlabeled data**, meaning there are no correct outputs provided during training. The algorithm is left on its own to explore the data, find hidden patterns, and group similar things together without any guidance. It is called "unsupervised" because there is no teacher telling the model whether its answers are right or wrong — the model must discover structure in the data entirely by itself. This makes it useful in situations where labeling data manually is expensive or impossible.

<img  width="500" height="fit-content" alt="image" src="https://github.com/user-attachments/assets/3d769f20-8d60-4580-bb76-d9e54c726077" />


**Two main types of problems it solves:**
- **Clustering** — grouping similar data points together based on their features (e.g., grouping customers by buying behavior).
- **Association** — finding rules that describe relationships between variables in large datasets (e.g., people who buy bread also tend to buy butter).

**Key characteristics:**
- Does not require labeled training data.
- The model finds patterns and structure on its own.
- Harder to evaluate because there is no known correct answer to compare against.

**Example:** Giving a model thousands of customer purchase records with no labels, and it automatically groups customers into segments like "frequent buyers" and "seasonal buyers," or discovers that customers who buy milk also tend to buy eggs.

### 3. Reinforcement Learning

Reinforcement learning is a type of machine learning where an **agent** learns to make decisions by interacting with an **environment** and receiving feedback in the form of **rewards or penalties**. The agent is not given any labeled data or told what the correct action is — instead, it learns by trial and error, trying different actions and observing what happens. Over time, the agent figures out which actions lead to the most reward and builds a strategy called a **policy** to maximize total reward. It is called "reinforcement" because good actions are reinforced (rewarded) and bad actions are discouraged (penalized).

**Key characteristics:**
- No labeled data is needed — learning happens through interaction.
- The agent focuses on maximizing **cumulative reward** over time, not just immediate reward.
- Learning can be slow because the agent must explore many possibilities.


**Example:** Training a computer to play chess — it plays thousands of games, receives a reward for winning and a penalty for losing, and gradually learns the best moves to make.


## 2. Inductive vs Deductive Learning

**Inductive Learning** is a type of learning where the model observes many specific examples and derives a general rule from them. It learns by looking at data and finding patterns, so the conclusion it reaches may not always be 100% certain but is based on evidence. For example, after seeing hundreds of cat images, the model concludes that an animal with pointy ears, fur, and whiskers is a cat.

**Deductive Learning** is a type of learning where the model starts with already known general rules and applies them to reach a specific conclusion. It does not need any training data — it simply follows logic. For example, given the rule "all mammals are warm-blooded" and the fact "a dog is a mammal," the model concludes that a dog is warm-blooded.

| Inductive Learning | Deductive Learning |
|---|---|
| It learns general rules from specific examples. | It applies known general rules to reach specific conclusions. |
| Direction is from specific to general. | Direction is from general to specific. |
| It is based on observation and data. | It is based on pre-defined logic and rules. |
| The conclusion may not always be 100% certain. | The conclusion is always logically certain. |
| It requires training data to learn patterns. | It does not require data — it works from given rules. |
| It is used in most machine learning algorithms. | It is used in expert systems and logic-based AI. |
| Example: learning to identify a cat from hundreds of labeled cat images. | Example: concluding a dog is warm-blooded from the rule that all mammals are warm-blooded. |

## 3. Machine Learning Framework

A machine learning framework is a structured model that describes how a learning system is built and how it operates. It breaks the entire learning process into distinct components, each playing a specific role in helping the system improve its performance over time. Understanding this framework helps in designing intelligent systems that can learn from experience and adapt to new situations.

<img  width="500" height="fit-content" alt="image" src="https://github.com/user-attachments/assets/bef28064-4571-43d7-b86e-435f4fc2d7fa" />


**Components:**

**Environment** is the external world or data source from which the system receives input. It provides the raw experiences, observations, or data that the learning system uses to learn from.

**Learning Element** is the component responsible for improving the system's knowledge by processing input from the environment. It updates the knowledge base based on what it observes and the feedback it receives.

**Performance Element** is the component that uses the current knowledge base to make decisions or take actions. It is the part of the system that actually performs the task the system was designed for.

**Knowledge Base** is the stored collection of facts, rules, and patterns that the system has learned so far. Both the learning element and the performance element depend on it — the learning element updates it and the performance element reads from it.

**Expert (Critic)** is the component that evaluates the performance of the system by comparing its outputs to the expected or correct results. It generates feedback or a performance signal that tells the learning element how well the system is doing, so it can improve.

**How they connect:**
- The **environment** sends data → the **performance element** acts on it → the **expert** evaluates the action → feedback goes to the **learning element** → it updates the **knowledge base** → the performance element improves.

## 4. Genetic Algorithms

A Genetic Algorithm (GA) is a search and optimization technique inspired by the process of **natural evolution and Darwin's survival of the fittest**. It starts with a population of candidate solutions and evolves them over multiple generations by applying biological operators like selection, crossover, and mutation. In each generation, better solutions are kept and weaker ones are discarded, gradually producing an optimal or near-optimal solution. It is especially useful when the search space is large and traditional methods are too slow or ineffective.

**Components / Steps:**

**Initialization** is the first step where a population of candidate solutions is generated randomly. Each individual in the population is called a **chromosome**, which represents one possible solution encoded as a string of bits or values. Each position in the chromosome is called a **gene**.

**Selection** is the process of choosing the fittest individuals from the current population to become parents for the next generation. A **fitness function** is used to evaluate how good each solution is — individuals with higher fitness scores have a greater chance of being selected.

**Crossover (Recombination)** is the process of combining two selected parent chromosomes to produce new offspring. A crossover point is chosen randomly and genes are swapped between the two parents, so each child inherits traits from both parents.

**Mutation** is the process of randomly changing one or more genes in a chromosome. It introduces new genetic variation into the population and prevents the algorithm from getting stuck at a local optimum.

**Termination** is the final step where the algorithm decides when to stop. The algorithm terminates when a satisfactory fitness level is reached, when the maximum number of generations is completed, or when the population shows no significant improvement over several generations.

**Applications / When to Use:**
- Used in **scheduling problems** such as timetable generation and resource allocation.
- Used in **route optimization** such as the Travelling Salesman Problem (TSP).
- Used in **engineering design** to find the best structure, shape, or configuration.
- Used in **neural network training** to optimize weights and architecture.
- Use GA when the problem has **no clear mathematical solution** but a fitness function can be defined.

## 5. Fuzzy Logic

Fuzzy Logic is a computing approach based on **degrees of truth** rather than the strict true or false (1 or 0) of classical logic. In real life, most things are not perfectly true or perfectly false — for example, a person can be "somewhat tall" or "very hot" rather than just tall or not tall. Fuzzy logic handles this by allowing values between 0 and 1, where 0 means completely false and 1 means completely true, and anything in between represents partial truth. It is widely used in systems that need to reason with imprecise or vague information.

**Characteristics:**
- It works with **partial truth values** between 0 and 1, not just true or false.
- It uses **linguistic variables** such as "hot," "cold," "fast," or "slow" to represent real-world concepts.
- It is **flexible and tolerant** of imprecise or noisy input data.
- It closely mimics **human reasoning**, making it suitable for systems that handle everyday decisions.
- It can handle problems where **mathematical models are difficult or impossible** to define precisely.

**When to Use:**
- Use when the problem involves **vague, imprecise, or linguistic information** (e.g., "temperature is high").
- Use when building **control systems** that mimic human judgment, such as air conditioners, washing machines, or braking systems.
- Use when a **precise mathematical model** of the system is hard to build.
- Use when the system needs to behave **smoothly across a range of inputs** rather than switching abruptly.

**When NOT to Use:**
- Do not use when the problem has a **clear and precise mathematical solution** already available.
- Do not use when inputs and outputs are **well-defined and crisp** with no ambiguity.
- Do not use when the system requires **very high accuracy and precision**, such as in scientific calculations.
- Do not use when **real-time performance is critical** and computational overhead is a concern.

### Fuzzy Logic Architecture

**1. Crisp Input**
The crisp input is the exact, precise numerical value that comes from the real world before any fuzzy processing happens. It is a normal measurement with no ambiguity, such as a temperature reading of 35°C or a speed of 60 km/h. This value alone cannot be used directly in fuzzy logic because fuzzy logic works with degrees, not exact numbers — so it must first be processed.

**2. Fuzzification**
Fuzzification is the process of converting the crisp input into fuzzy values using **membership functions**. A membership function defines how much a given input value belongs to a fuzzy set, and it returns a value between 0 and 1. For example, the temperature 35°C might get a membership of 0.8 in the "hot" set and 0.2 in the "warm" set. A single crisp input can partially belong to multiple fuzzy sets at the same time, which is what makes fuzzy logic flexible and human-like.

**3. Rule Base (Knowledge Base)**
The rule base is a collection of **IF-THEN rules** written by a human expert that describes how the system should behave. Each rule connects fuzzy input conditions to a fuzzy output action. For example: *"IF temperature is hot AND humidity is high THEN fan speed is fast."* The rules capture expert knowledge in simple linguistic form. The quality of the rule base directly determines the quality of the system — if the rules are poorly written, the system will behave poorly.

**4. Inference Engine**
The inference engine is the brain of the fuzzy system. It reads the fuzzified input values, checks them against every rule in the rule base, and determines which rules are triggered and to what degree. It then combines all the triggered rules to produce a **fuzzy output set**. For example, if "temperature is hot" with degree 0.8 and the rule says "IF hot THEN fan = fast," the inference engine activates the "fast" output to a degree of 0.8. This process is called **Mamdani inference** when using the min-max method.

**5. Defuzzification**
Defuzzification is the process of converting the fuzzy output set produced by the inference engine back into a single crisp numerical value that can actually be used to control something. The most common method is the **Centroid Method (Center of Gravity)**, which calculates the center point of the combined fuzzy output area. For example, if the fuzzy output represents "fan speed is fast," defuzzification might convert that into a precise value like 80% fan speed. Without defuzzification, the system cannot give a usable real-world command.

**6. Crisp Output**
The crisp output is the final precise numerical value that comes out of the system after defuzzification. This is the actual control signal sent to the physical device, such as setting the fan motor to exactly 80% speed. It is a normal number again — but it was derived through a fuzzy reasoning process that mimics human judgment.


## 6. ID3 Algorithm

ID3 (Iterative Dichotomiser 3) is a **decision tree learning algorithm** that builds a tree by selecting the best attribute to split the data at each step. It was developed by Ross Quinlan. The best attribute is chosen using **Information Gain** — the attribute that reduces the most uncertainty (entropy) in the dataset is selected as the splitting node. It keeps splitting until all data in a branch belongs to the same class or no attributes are left.

**Key Concepts:**

**Entropy** measures the impurity or uncertainty in a dataset. If all examples belong to one class, entropy is 0 (pure). If examples are equally split, entropy is 1 (maximum uncertainty).

$$E(S) = -\sum p_i \log_2 p_i$$

**Information Gain** measures how much an attribute reduces entropy after splitting. The attribute with the highest information gain is chosen as the decision node.

$$IG(S, A) = E(S) - \sum \frac{|S_v|}{|S|} \cdot E(S_v)$$

**Algorithm Steps:**

1. Calculate the entropy of the entire dataset.
2. For each attribute, calculate the information gain.
3. Select the attribute with the highest information gain as the root/decision node.
4. Split the dataset into subsets based on that attribute.
5. Repeat steps 1–4 recursively for each subset.
6. Stop when all examples in a node belong to the same class, or no attributes remain.

**Limitations:**
- It favors attributes with more values, which can lead to overfitting.
- It does not handle continuous or missing values well.
- It builds the entire tree without pruning, so the tree can become too large.
