# Unswitched (broadcast) network
In an **unswitched (broadcast) network**, every device receives all transmitted data regardless of the intended destination — the data is broadcast to all nodes and each device decides whether to accept or ignore it. A basic Ethernet LAN is an example.

# Switched network
In a **switched network**, a dedicated path or logical connection is established between sender and receiver, and data is directed only toward the intended destination through intermediate switching nodes. This makes switched networks more efficient, scalable, and suitable for large networks like the internet.

## 1. Circuit Switching (Circuit Switched Network)

In **circuit switching**, a dedicated physical path is established between sender and receiver before any data is transmitted, and this path remains reserved for the entire duration of the communication. The intermediate switches along the path make no routing decisions during transfer — everything is decided at connection setup. The telephone network (PSTN) is the classic example.

### Three Phases of Circuit Switching

### 1. Setup Phase
Before communication begins, a dedicated end-to-end path is set up through all intermediate switches. Resources such as bandwidth and switch capacity are reserved along this path for the entire session.

### 2. Data Transfer Phase
Once the path is established, data flows continuously along the reserved path. Since the path is dedicated, there is no delay or congestion during transmission — the full bandwidth is available to the communicating parties throughout.

### 3. Termination Phase
After communication ends, the reserved path is released and the resources are freed for use by other connections.

### Advantages
- Guaranteed bandwidth and consistent performance throughout the session.
- No delay once the connection is established — ideal for real-time communication like voice calls.
- Simple data transfer since routing decisions are made only once at connection setup.

### Disadvantages
- Resources are wasted during idle periods — the path is reserved even when no data is being sent.
- Connection setup takes time before communication can begin.
- Inefficient for bursty data traffic like internet browsing where data is not sent continuously.

## 2. Packet Switching

In **packet switching**, data is broken into smaller units called **packets**, each of which is routed independently through the network. Unlike circuit switching, no dedicated path is reserved — each packet finds its own way to the destination and the packets are reassembled in order at the receiver end. The internet is the primary example of packet switching.

### Two Approaches to Packet Switching

### Datagram Switching


<img width="933" height="541" alt="image" src="https://github.com/user-attachments/assets/758a7190-6ef4-47d1-83e9-8bf0785e9959" />


In **datagram switching**, the message is divided into packets and each packet is treated as a completely independent unit. Every packet carries the full source and destination address in its header, and each intermediate router examines this address and makes its own independent routing decision based on its current routing table. Because routing decisions are made individually at each hop, different packets of the same message may travel through entirely different paths and arrive at the destination out of order or even at different times. The receiver is responsible for reordering and reassembling the packets into the original message. This approach is highly fault tolerant — if a router or link fails, subsequent packets are simply routed around it. The internet (IP network) is the primary example of datagram switching.

### Virtual Circuit Switching


<img width="620" height="336" alt="image" src="https://github.com/user-attachments/assets/3cea695d-263e-4c86-afa4-25a9ec8eacfa" />


In **virtual circuit switching**, a logical path called a **virtual circuit** is established between sender and receiver before data transfer begins. During setup, each intermediate router records the path in its routing table and assigns a **Virtual Circuit Identifier (VCI)** to the connection. All packets then follow this same predetermined path in order, carrying only the VCI in their header instead of the full address — making headers smaller and routing faster. Unlike circuit switching, no physical resources are reserved — the path is purely logical. When the session ends, the virtual circuit is torn down and the entries are removed from router tables. ATM and Frame Relay are classic examples.

| | Datagram Switching | Virtual Circuit Switching |
|---|---|---|
| **Path** | No fixed path — each packet routed independently. | Fixed logical path established before transfer. |
| **Addressing** | Each packet carries full source and destination address. | Each packet carries only a small VCI. |
| **Routing decision** | Made at every router for every packet. | Made once during connection setup. |
| **Packet order** | Packets may arrive out of order. | Packets always arrive in order. |
| **Fault tolerance** | High — packets reroute around failures automatically. | Low — if the virtual circuit fails, the session breaks. |
| **Setup delay** | No setup needed — transmission starts immediately. | Setup delay before data transfer begins. |
| **Overhead** | Higher per-packet overhead due to full address. | Lower per-packet overhead due to small VCI. |
| **Resource reservation** | No reservation — fully dynamic. | No physical reservation but logical path is fixed. |
| **Example** | Internet (IP) | ATM, Frame Relay, X.25 |

### Advantages
- Efficient use of network resources since bandwidth is shared among multiple users.
- More fault tolerant — if one route fails, packets can take alternate paths (datagram).
- Better suited for bursty data traffic like internet browsing.

### Disadvantages
- Variable delay (**jitter**) since packets may wait in queues at each router.
- Packets can arrive out of order and must be reassembled.
- Overhead is added since each packet carries addressing and control information.

| | Circuit Switching | Packet Switching |
|---|---|---|
| **Path** | Dedicated physical path | No dedicated path |
| **Resource reservation** | Reserved for entire session | Shared dynamically |
| **Routing** | Decided once at setup | Decided per packet (datagram) |
| **Delay** | No delay during transfer | Variable delay (jitter) |
| **Efficiency** | Wasteful during idle periods | Efficient for bursty traffic |
| **Example** | PSTN telephone network | Internet |
