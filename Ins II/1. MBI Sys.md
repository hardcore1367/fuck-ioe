# MBI System

## 1. Features/Benefits

- **Programmability** — system behavior can be changed by simply modifying software without changing hardware, making it flexible and reusable for different tasks
- **High accuracy and precision** — digital processing eliminates errors caused by analog component drift and variations, giving more reliable measurements
- **Multi-channel capability** — can acquire and process data from multiple sensors simultaneously using a multiplexer
- **Data storage and retrieval** — measured data can be stored in memory for later use
- **Real time processing** — can process input signals and respond instantly
- **Communication capability** — can transmit data to computers or other systems via serial or parallel communication for remote monitoring
- **Compact and low cost** — microprocessor replaces large analog circuits with a small chip, reducing size, power consumption and overall cost
- **Reduced human effort** — automation of measurement, calculation and reporting reduces dependency on skilled operators

## 3. Interfacing with Microprocessor
Interfacing is the process of connecting external devices such as sensors, displays, memory and communication ports to a microprocessor so that data can be transferred between them.

### 1. PC Interfacing Techniques
Methods used to connect external devices or circuits to a PC for data acquisition and control.

**1. I/O Bus**
A bus is a set of shared communication lines that connect the microprocessor to memory and external devices. It carries data, address and control signals between them. All devices share the same bus and a specific device is selected using address decoding.

- Data bus carries actual data
- Address bus selects which device to access
- Control bus carries read and write signals
- Examples: PCI and PCIe slots used for data acquisition cards inside PC

**2. Parallel Port**
A parallel port sends 8 bits of data at the same time through 8 separate wires. It was commonly used to connect printers and simple external devices to a PC but is now mostly replaced by USB.

- Fast for short distances but cable length limited to about 2 meters
- Simple to use for basic digital input and output
- Now mostly obsolete in modern systems


**3. Serial Port (RS232)**
A serial port sends data one bit at a time through a single wire. It is slower than parallel port but works over longer distances and is still used in many industrial and embedded systems.

- Uses TX line for transmit and RX line for receive
- Simple, cheap and reliable
- Commonly used with industrial sensors and embedded devices

**4. USB (Universal Serial Bus)**
USB is the most common modern interface for connecting devices to a PC. It is fast, easy to use and supports plug and play meaning the device is automatically recognized when connected.

- Much faster than serial port
- Can connect multiple devices using a USB hub
- Provides 5V power to connected devices through the cable
- Used for microcontrollers, sensors and data loggers

### 2. I/O Interfacing Techniques

Methods used to transfer data between microprocessor and external devices.


**1. Programmed I/O (Polling)**
The microprocessor continuously checks the status of a device in a loop to see if it is ready to send or receive data.

- Simple to implement
- Processor wastes time continuously checking the device
- Not suitable for multiple devices or time critical tasks

**2. Interrupt Driven I/O**
The device sends an interrupt signal to the microprocessor when it is ready. The processor stops its current task, serves the device and then returns back to its previous task.

- More efficient than polling as processor does not waste time waiting
- Processor can perform other tasks while waiting for device
- Suitable for time critical systems

**3. Direct Memory Access (DMA)**
Data is transferred directly between the device and memory without involving the microprocessor. A special chip called DMA controller manages the transfer.

- Fastest method of data transfer
- Processor is completely free during transfer
- Used for high speed devices such as ADC, disk drives and cameras

## 4. Direct Memory Access (DMA)

DMA is a method of data transfer where data moves directly between a device and memory without involving the microprocessor. A dedicated chip called the DMA controller manages the entire transfer, freeing the processor to do other tasks.

**Why DMA is needed**
- In polling and interrupt driven I/O, every byte of data passes through the processor which slows down transfer
- For high speed devices, this is too slow
- DMA allows large amounts of data to be transferred at full bus speed without processor involvement

**How DMA Works**
1. Device requests data transfer by sending a signal to DMA controller
2. DMA controller sends a **bus request** to the processor asking to take control of the bus
3. Processor finishes current instruction and sends **bus grant** signal, then releases the bus
4. DMA controller takes control of address bus, data bus and control bus
5. DMA controller transfers data directly between device and memory
6. After transfer is complete, DMA controller releases the bus and informs processor via interrupt

**Active Cycle** — DMA controller takes control of the bus and transfers data between device and memory. Processor cannot use the bus during this time.

**Idle Cycle** — DMA controller is not transferring data and bus is free for the processor to use normally.


**Advantages**
- Fastest method of data transfer
- Processor is free to perform other tasks during transfer
- Suitable for large and continuous data transfers

**Disadvantages**
- Requires dedicated DMA controller chip, increasing hardware cost
- More complex to implement than polling or interrupt driven I/O
- Bus conflict can occur if not properly managed

## 5. I/O Mapped I/O vs Memory mapped I/O
| | Memory Mapped I/O | Port Mapped I/O |
|---|---|---|
| Address space | Shares same address space as memory | Separate address space for I/O |
| Instructions | Same read/write instructions as memory | Special IN and OUT instructions |
| Memory space | Reduces available memory address space | Memory space is not affected |
| Complexity | Simpler, no special instructions needed | Requires separate I/O instructions |
| Used in | Most microcontrollers, ARM processors | Intel x86 based systems |
| Speed | Slightly faster as memory instructions are optimized | Slightly slower due to separate I/O instructions |

## 6. Microprocessor vs Microcontroller
| Microprocessor | Microcontroller |
|---|---|
| Only the CPU is present on the chip, requiring external memory and I/O components | CPU, memory and I/O are all integrated on a single chip |
| Higher cost due to need for external components | Lower cost as everything is on one chip |
| Larger system size due to external components | Compact size suitable for embedded applications |
| Higher processing speed | Lower processing speed |
| Higher power consumption | Lower power consumption |
| Requires external RAM, ROM and I/O chips | Mostly self contained with built in peripherals |
| Used in personal computers, laptops and high performance systems | Used in embedded systems, home appliances and sensors |
| Example: Intel Core, AMD Ryzen | Example: Arduino, PIC, STM32 |


## 7. Selection Criteria for Embedded System Design

- **Processing power** — choose microprocessor if heavy computation is needed, microcontroller for simple tasks
- **Cost** — microcontroller is cheaper for simple dedicated applications
- **Power consumption** — microcontroller consumes less power, preferred for battery operated devices
- **Size** — microcontroller gives compact design as everything is on one chip
- **I/O requirements** — microcontroller has built in GPIO, ADC, timers and communication ports
- **Development time** — microcontroller is faster to develop with as less external hardware is needed
- **Real time requirements** — microcontrollers are better for real time control tasks
- **Memory requirement** — if large memory is needed, microprocessor with external RAM is preferred

## 8. ISA Bus vs PCI Bus

| ISA Bus | PCI Bus |
|---|---|
| Stands for Industry Standard Architecture | Stands for Peripheral Component Interconnect |
| Data bus width is 8 or 16 bit | Data bus width is 32 or 64 bit |
| Clock speed is 8 MHz | Clock speed is 33 or 66 MHz |
| Does not support bus mastering | Supports bus mastering |
| Does not support plug and play | Supports plug and play |
| Used in older PCs and simple peripherals | Used in modern expansion cards |
