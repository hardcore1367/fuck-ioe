# Serial Interfacing

## 3.1 Serial Data Transmission

In serial data transmission (SDT), data is sent one bit at a time over the transmission channel. It is slower compared to parallel transmission.

### 1. Advantages of Serial over Parallel

- Longer data transmission is possible.
- Voltage loss is not a problem in SDT.
- It requires less number of wires and is cheaper to transmit data.
- Crosstalk is not an issue because there are fewer conductors compared to parallel transmission.

### 2 Synchronous Serial Data Transmission

Data is sent as a continuous stream of bits with no gaps between characters. A **shared clock signal** is used to synchronize the sender and receiver so both know exactly when each bit starts and ends. Data is grouped into **frames or blocks**, and special **sync characters** are sent at the beginning to establish synchronization. Because there are no start/stop bits per character, it is more efficient and faster than asynchronous transmission.

**Advantages:** Faster, less overhead
**Disadvantages:** Complex hardware, expensive, sensitive to clock loss


## 3 Asynchronous Serial Data Transmission

Data is sent **one character at a time** with no shared clock. Instead, each character is framed with a **start bit (0)** at the beginning and one or two **stop bits (1)** at the end. The receiver detects the start bit and uses its own internal clock to sample the remaining bits. Between characters, the line stays in **idle state (logic 1)**. This makes it simple to implement but adds per-character overhead.

Each character frame consists of:
- **1 start bit**
- **7 or 8 data bits**
- **1 parity bit**
- **1 or 2 stop bits**

**Advantages:** Simple, cheap, no clock sync needed
**Disadvantages:** Slower, not ideal for bulk transfer

## 4. Synchronous vs Asynchronous Serial Transmission

| Synchronous | Asynchronous |
|---|---|
| Uses a shared clock signal between sender and receiver | No shared clock; receiver uses its own internal clock |
| Data is sent as a continuous stream in blocks or frames | Data is sent one character at a time |
| Sync characters sent at the start to establish timing | Each character framed with start and stop bits |
| No start/stop bits per character, so less overhead | 1 start bit, 7/8 data bits, 1 parity bit, 1/2 stop bits per character |
| Faster and more efficient for large data transfer | Slower due to per-character framing overhead |
| Complex and expensive hardware required | Simple and cheap to implement |
| Loss of sync corrupts the entire block | Error in one character does not affect others |

## 3.3 Errors in Serial Data Transmission

- **Framing Error** — Occurs when the receiver expects a stop bit (logic 1) but receives a 0 instead. This means the receiver's clock has drifted out of sync with the sender, causing the frame boundaries to misalign.

- **Break Error** — Occurs when the transmission line stays at logic 0 (space/low) continuously for longer than a full character frame duration. This is usually caused by a broken wire, disconnected cable, or a deliberate break signal sent by the transmitter.

- **Overrun Error** — Occurs when the sender transmits the next character before the receiver has finished reading the previous one from its buffer. The old data gets overwritten before it is processed, resulting in data loss.

- **Parity Error** — Occurs when the number of 1s in the received data bits does not match the expected parity (odd or even). This indicates that one or more bits were flipped or corrupted during transmission due to noise on the line.
