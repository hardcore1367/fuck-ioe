## 1. 8051 Important / Operational Features

The **8051** is an 8-bit microcontroller developed by Intel. It has everything needed for embedded control on a single chip.

- 8-bit CPU
- 4KB internal ROM, 128 bytes internal RAM
- 4 I/O ports (P0, P1, P2, P3) — 32 I/O pins total
- Two 16-bit timers/counters (Timer 0, Timer 1)
- One full-duplex UART (serial communication)
- 6 interrupt sources (2 external, 2 timer, 1 serial)
- 64KB external ROM and 64KB external RAM expandable
- Single power supply (+5V), clock frequency up to 12 MHz
- Boolean processor — can perform bit-level operations

---

## 2. 8051 PORT Description

**Port 0 (P0):**
- 8-bit bidirectional I/O port
- Also used as **address/data bus** when external memory is connected
- Needs external pull-up resistors (open drain)

**Port 1 (P1):**
- 8-bit bidirectional I/O port
- **Dedicated general purpose I/O** — no alternate function
- Has internal pull-up resistors

**Port 2 (P2):**
- 8-bit bidirectional I/O port
- Also used as **high-order address bus (A8–A15)** during external memory access

**Port 3 (P3):**
- 8-bit bidirectional I/O port
- Each pin has **alternate special function:**

| Pin | Alternate Function |
|---|---|
| P3.0 | RXD (Serial input) |
| P3.1 | TXD (Serial output) |
| P3.2 | INT0 (External interrupt 0) |
| P3.3 | INT1 (External interrupt 1) |
| P3.4 | T0 (Timer 0 input) |
| P3.5 | T1 (Timer 1 input) |
| P3.6 | WR (Write strobe) |
| P3.7 | RD (Read strobe) |

## 3. 8051 Addressing Modes

**Addressing mode** defines how the operand (data) is accessed in an instruction.

**A) Immediate Addressing**
Data is given directly in the instruction.
`MOV A, #25H` → load 25H directly into A

**B) Register Addressing**
Data is in one of the registers (A, B, R0–R7).
`MOV A, R0` → copy R0 into A

**C) Direct Addressing**
Data is at a specific RAM address given in instruction.
`MOV A, 30H` → load content of address 30H into A

**D) Indirect Addressing**
Address of data is stored in a register (R0 or R1).
`MOV A, @R0` → use R0 as pointer to get data

**E) Indexed Addressing**
Used for accessing lookup tables. Address = base register + offset.
`MOVC A, @A+DPTR`

## 4. 8051 Data Memory Organization

8051 has **128 bytes of internal RAM** organized as:

| Address | Region | Use |
|---|---|---|
| 00H – 1FH | **Register Banks** | 4 banks × 8 registers (R0–R7) |
| 20H – 2FH | **Bit-Addressable Area** | 128 individually addressable bits |
| 30H – 7FH | **General Purpose RAM** | Scratch pad / stack area |

- **Stack** grows upward, default Stack Pointer (SP) starts at 07H
- **Special Function Registers (SFRs)** are at 80H–FFH (not part of 128 bytes)
- SFRs include: ACC, B, PSW, SP, DPTR, TCON, TMOD, SCON, IE, IP, ports

## 5. 7-Segment Display — Types & Configuration

A **7-segment display** has 7 LED segments (a–g) + 1 decimal point, used to display digits 0–9.

**Two types:**

**A) Common Anode (CA):**
- All anodes connected together to Vcc
- Segment turns ON by sending **logic 0** to that segment
- Active LOW

**B) Common Cathode (CC):**
- All cathodes connected together to GND
- Segment turns ON by sending **logic 1** to that segment
- Active HIGH

| Feature | Common Anode | Common Cathode |
|---|---|---|
| Common pin | +Vcc | GND |
| To turn ON segment | Send 0 | Send 1 |
| Logic | Active LOW | Active HIGH |

**Segment codes for digit "0" (Common Cathode):** segments a,b,c,d,e,f ON → 0x3F

## 6. Microcontroller vs Microprocessor

| Feature | Microcontroller | Microprocessor |
|---|---|---|
| Definition | CPU + RAM + ROM + I/O on one chip | Only CPU on chip |
| External components | Minimal needed | Needs external RAM, ROM, I/O |
| Cost | Low | High |
| Speed | Lower | Higher |
| Power consumption | Low | High |
| Application | Embedded systems (washing machine, car) | General purpose (PC, laptop) |
| Example | 8051, PIC, AVR | Intel 8086, Core i5 |
