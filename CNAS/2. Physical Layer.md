The physical layer (Layer 1) is the lowest level of the OSI model. It establishes the hardware foundation of a network by converting digital data into raw electrical, optical, or radio signals to transmit unstructured data streams (bits) between devices.

## 2. Switching

Switching is the process of forwarding data (messages or packets) from a source device to a destination device through a network of interconnected nodes. It ensures that data is directed through the optimal path to arrive at its intended location efficiently.

### 2.1 Circuit Switching

Circuit switching is a communication method in computer networks where a dedicated path is established between two devices before any data is transmitted. This reserved path remains fixed throughout the entire communication session.

<img width="1024" height="427" alt="image" src="https://github.com/user-attachments/assets/5ecbc082-a0ec-4d54-b383-a533ef6da992" />


1. **Setup Phase:** A dedicated circuit is established from the sender to the receiver. The network reserves all the necessary resources like bandwidth along the path before communication begins.
2. **Data Transfer phase:** Actual data is sent through the established circuit. Since the path is already fixed and resources are reserved, data flows continuously and in order without any delay.
3. **Termination phase:** Once the communication is complete, the circuit is released and all the reserved resources are freed up for other connections.

### 2.2 Message Switching

Message switching is a communication method in computer networks where the entire message is sent from the source to the destination through intermediate nodes. Unlike circuit switching, no dedicated path is established. Instead, each intermediate node receives the complete message, stores it temporarily, and then forwards it to the next node toward the destination. This is known as the store-and-forward technique. The message travels hop by hop until it reaches the final destination.

Some drawbacks of message switching are that since each intermediate node stores the entire message before forwarding, it causes significant delays especially for large messages. It also requires large storage capacity at each intermediate node to hold messages temporarily. Lastly, if an intermediate node fails, the message stored in it can be lost, making it unreliable.

### 2.2 Packet Switching

Packet switching is a communication method in computer networks where data is broken down into smaller units called packets before transmission. Each packet is sent independently through the network and may take different paths to reach the destination. At the destination, all packets are reassembled to form the original message. No dedicated path is reserved, and network resources are shared among multiple users.

#### 2.2.1 Datagram switching

Datagram switching is a connectionless approach of packet switching in computer networks where each packet is treated as an independent unit and routed separately through the network. The data is divided into smaller units called packets, and each packet contains important information such as the source address, destination address, and sequence number. Since each packet is independent, they can take different routes to reach the same destination depending on the network conditions and router decisions at that moment.

In datagram switching, there is no need to establish a dedicated path before transmission begins. Each router in the network makes its own decision about where to forward the packet based on its routing table. This means that even two consecutive packets belonging to the same message may travel through completely different paths. Because of this, packets may arrive at the destination out of order, late, or sometimes not at all. It is the responsibility of the destination to collect all the packets, reorder them using the sequence numbers, and reassemble them into the original message.

One of the key features of datagram switching is its flexibility. Since no fixed path is used, if one route in the network becomes congested or a node fails, packets can be rerouted through an alternative path automatically. This makes the network more robust and fault tolerant. However, the overhead is higher because each packet must carry its own addressing information and each router must make individual routing decisions for every packet passing through it.

#### 2.2.2 Virtual circuit switching

Virtual circuit switching is a connection-oriented approach of packet switching in computer networks where a logical path is established between the sender and receiver before any data transmission begins. This logical path is called a virtual circuit. Unlike datagram switching, all packets belonging to the same message follow the same fixed path in order from source to destination.

Virtual circuit switching works in three phases. In the setup phase, a virtual circuit is established and each node along the path is informed about the route. A virtual circuit identifier is assigned to the connection. In the data transfer phase, all packets follow the same pre-defined path using the virtual circuit identifier instead of carrying the full destination address in each packet, which reduces overhead. In the termination phase, once communication is complete, the virtual circuit is released and resources are freed.

Since all packets travel through the same path, they arrive at the destination in order, which makes reassembly easier and faster. The network is also more predictable in terms of delay and performance compared to datagram switching. However, if any node along the virtual circuit fails, the entire communication is disrupted and a new virtual circuit must be established. Also, the initial setup time is required before data transmission can begin, which adds a small delay at the start of communication.

### Circuit Switching vs Packet Switching

| Circuit Switching | Packet Switching |
|---|---|
| A dedicated path is established between sender and receiver before data transmission begins | No dedicated path is established, each packet is routed independently through the network |
| Resources are fully reserved throughout the entire communication session | Resources are shared among multiple users and used only when data is being transmitted |
| Fixed bandwidth is allocated to the connection even if no data is being sent | Bandwidth is used on demand and not wasted during idle periods |
| There is a setup delay at the beginning but no delay during actual data transmission | There is no setup delay but each packet experiences transmission delay at every node |
| Data is sent as a continuous stream through the fixed dedicated path | Data is broken down into smaller packets and each packet may take a different route to reach the destination |
| It is less efficient because resources remain reserved even when the channel is idle | It is more efficient because network resources are shared dynamically among all users |
| The traditional telephone network is a real world example of circuit switching | The internet is a real world example of packet switching |

### Datagram Switching vs Virtual Circuit Switching

| Datagram Switching | Virtual Circuit Switching |
|---|---|
| It is a connectionless approach where no path is established before data transmission begins | It is a connection-oriented approach where a logical path called virtual circuit is established before data transmission begins |
| Each packet is treated independently and may take a different route to reach the destination | All packets belonging to the same message follow the same fixed logical path from source to destination |
| Each packet carries the full source and destination address which increases overhead | Packets carry only the virtual circuit identifier instead of full address which reduces overhead |
| Packets may arrive at the destination out of order since they travel through different paths | Packets always arrive at the destination in order since they all follow the same path |
| There is no setup delay as no connection is established before transmission | There is a setup delay at the beginning as a virtual circuit must be established before transmission |
| If a node fails, packets can be rerouted through an alternative path automatically making it fault tolerant | If any node along the virtual circuit fails, the entire communication is disrupted and a new virtual circuit must be established |
| It is more flexible as routing decisions are made independently at each router for every packet | It is less flexible as all packets must follow the same pre-defined logical path throughout the session |

### 3.1 Frame Relay

Frame relay is a wide area network protocol that operates on the principle of virtual circuit switching. It uses virtual circuits to establish a logical connection between two endpoints in the network.

Switched Virtual Circuit (SVC) is a type of virtual circuit in frame relay that is dynamically established on demand whenever communication is needed and released after the communication is complete, similar to a phone call. The establishment of a switched virtual circuit in frame relay works in three stages.

In the call setup stage, when a device wants to communicate, it sends a call request signal through the network. The network then finds a suitable path and establishes a virtual circuit between the source and destination. A Data Link Connection Identifier (DLCI) is assigned to identify the virtual circuit.

In the data transfer stage, once the virtual circuit is established, data is transmitted in the form of frames through the logical path. All frames follow the same virtual circuit path using the DLCI number. The network forwards each frame from one node to the next until it reaches the destination.

In the call termination stage, once the communication is complete, either the sender or receiver sends a disconnection signal to release the virtual circuit. The DLCI is freed and all the resources reserved for that virtual circuit are released and made available for other connections.

### 3.2 X.25 Virtual circuit

X.25 is a wide area network protocol that defines how devices connect to a packet switched network. It is one of the oldest packet switching protocols and operates across three layers which are the physical layer, data link layer, and network layer. X.25 uses virtual circuits to establish a connection between two endpoints before data transmission begins, and it provides reliable and error-free delivery of data through extensive error checking and flow control at every node in the network.

Virtual Circuit Operation in X.25 works in three phases. In the call setup phase, a virtual circuit is established between the source and destination and a logical channel number is assigned to identify the connection. In the data transfer phase, data is transmitted in the form of packets through the established virtual circuit and all packets follow the same path. In the call clearing phase, once communication is complete, the virtual circuit is terminated and the logical channel number is released.

Packet Format of X.25 consists of three main fields. The first is the header which contains the logical channel number and control information. The second is the data field which carries the actual data being transmitted. The third is the error checking field which ensures that the data is received correctly without any errors.

### X.25 vs Frame Relay

| X.25 | Frame Relay |
|---|---|
| X.25 performs error checking and flow control at every node in the network which makes it slower | Frame relay performs error checking only at the endpoints and not at intermediate nodes which makes it faster |
| X.25 is designed for analog and noisy transmission lines where errors are more common | Frame relay is designed for digital and reliable transmission lines where errors are rare |
| X.25 has higher overhead due to extensive error checking and control information at each node | Frame relay has lower overhead as it eliminates node by node error checking |
| X.25 operates at a lower speed and is suitable for low bandwidth networks | Frame relay operates at a much higher speed and is suitable for high bandwidth networks |
| X.25 uses a three layer protocol stack including physical, data link, and network layer | Frame relay operates mainly at the data link layer which simplifies its operation |
| X.25 is an older protocol and is largely outdated in modern networks | Frame relay is a more modern protocol and was widely used before being replaced by newer technologies |

## 4. Multiplexing

Multiplexing is a method of combining multiple signals or data streams into a single signal for transmission over a shared medium, optimizing bandwidth and efficiency.

It is essential in data communications because physical links are expensive — multiplexing maximizes their utilization by combining many low-bandwidth signals into one high-bandwidth channel. A device called a **multiplexer (MUX)** combines signals at the sender side, and a **demultiplexer (DEMUX)** separates them back at the receiver side.

### 1. Frequency Division Multiplexing (FDM)

Frequency Division Multiplexing (FDM) is an analog multiplexing technique where the total bandwidth of a transmission medium is divided into smaller frequency bands, each assigned to a different signal. It is widely used in radio broadcasting, cable TV, and telephone systems. All signals travel simultaneously — they are separated by frequency, not by time.


<img width="1024" height="768" alt="image" src="https://github.com/user-attachments/assets/aac128bc-20f9-4925-a341-6a116a21a707" />


**Working (Multiplexing Side)**

Each input signal is a baseband signal — it originally occupies a low-frequency range starting near 0 Hz. Before combining, each signal is modulated (shifted) onto a unique carrier frequency so they occupy non-overlapping bands.

```
Signal 1 (baseband) ──► Modulate onto f₁ ──┐
Signal 2 (baseband) ──► Modulate onto f₂ ──┤──► MUX ──► Composite Signal ──► Link
Signal 3 (baseband) ──► Modulate onto f₃ ──┘
```

**Working (Demultiplexing)**

At the receiver, the composite signal arrives containing all channels stacked in frequency. The DEMUX separates them using bandpass filters — one per channel.

```
                    ┌── BPF(f₁) ──► Demodulate ──► Signal 1
Composite Signal ───┤── BPF(f₂) ──► Demodulate ──► Signal 2
                    └── BPF(f₃) ──► Demodulate ──► Signal 3
```

### 2. Wavelength Division Multiplexing (WDM)

**Wavelength Division Multiplexing (WDM)** is the optical equivalent of FDM, designed specifically for **fiber optic cables**. Instead of dividing frequency bands, it combines multiple light beams of different **wavelengths (colors)** onto a single fiber. Since different wavelengths of light do not interfere with each other, each wavelength acts as an independent channel carrying its own data stream.

**Working**

<img width="589" height="356" alt="image" src="https://github.com/user-attachments/assets/fb7e7695-e18e-4ef1-a822-de0795df1a41" />


Each input signal is carried by a different wavelength of light (λ₁, λ₂, λ₃...). A **prism or optical multiplexer** combines all these wavelengths into a single composite light beam and sends it through one fiber. At the receiver, another prism/demultiplexer splits the beam back into individual wavelengths.

The multiplexing and demultiplexing is done using **optical prisms** or **diffraction gratings** — purely optical components with no electronic conversion needed.

### 3. Time Division Multiplexing (TDM)

Time Division Multiplexing (TDM) is a digital multiplexing technique where multiple signals share a single link by taking turns — each signal gets the full bandwidth of the link but only for a brief, repeating time slot. Unlike FDM which divides frequency, TDM divides time. It is used in digital telephone networks, ISDN, and other digital communication systems.

**Working**

<img width="3400" height="1843" alt="image" src="https://github.com/user-attachments/assets/c42deb81-a7ef-4c9c-8729-92741a4f4abf" />


The link time is divided into a repeating cycle called a frame. Each frame is split into fixed time slots, one per input channel. Every channel gets its slot in every frame — whether it has data to send or not.

```
Frame 1                         Frame 2
┌────┬────┬────┬────┐           ┌────┬────┬────┬────┐
│ S1 │ S2 │ S3 │ S4 │           │ S1 │ S2 │ S3 │ S4 │ ──► Link
└────┴────┴────┴────┘           └────┴────┴────┴────┘
 slot slot slot slot
```

The MUX samples each input in round-robin order, placing one chunk per slot. The DEMUX at the receiver reads each slot and routes it to the correct output channel based on slot position. Synchronization is critical — sender and receiver must agree on frame timing.

#### 1. Synchronous TDM

In Synchronous TDM, each channel is assigned a fixed, dedicated time slot in every frame regardless of whether it has data to send. If a channel is idle, its slot is sent empty (wasted).

- Slot assignment is static and predetermined.
- Simple to implement — no overhead needed to identify which channel owns which slot (position implies identity).
- Inefficient when many channels are idle because empty slots still consume link capacity.

```
Frame: | S1(data) | S2(empty) | S3(data) | S4(empty) |
                    ↑ wasted slot
```

#### 2. Statistical TDM (Asynchronous TDM)

In Statistical TDM, time slots are allocated dynamically — only to channels that actually have data to send. Idle channels get no slot in that frame.

- Each slot carries a header (address/identifier) so the DEMUX knows which channel the data belongs to.
- The number of slots per frame varies based on demand.
- Link capacity is used only for real data — no empty slots.

```
Frame: | S1(data) | S3(data) | S1(data) | S3(data) |
         ↑ header tells DEMUX: "this belongs to channel 1"
```

### FDM vs TDM vs WDM

| | FDM | TDM | WDM |
|---|---|---|---|
| 1 | It divides bandwidth by frequency. | It divides bandwidth by time slots. | It divides bandwidth by wavelength of light. |
| 2 | It carries analog signals. | It carries digital signals. | It carries optical (light) signals. |
| 3 | It works over copper wire or radio. | It works over copper or fiber. | It requires fiber optic only. |
| 4 | All channels are active simultaneously. | Channels take turns using the medium. | All wavelengths are active simultaneously. |
| 5 | It separates channels using bandpass filters. | It separates using time slot position or headers. | It separates using optical prisms or diffraction gratings. |
| 6 | Idle channels waste bandwidth. | Idle slots waste bandwidth (sync) / no waste (statistical). | Idle wavelengths waste bandwidth. |
| 7 | Complexity is moderate. | Complexity is low (sync) or high (statistical). | Complexity is high due to precise optical equipment. |
| 8 | Capacity is moderate. | Capacity is moderate. | Capacity is extremely high — DWDM reaches Tbps. |
| 9 | Used in radio, cable TV, and DSL. | Used in phone networks, ISDN, and internet. | Used in fiber backbone and undersea cables. |

## 5. Integrated Services Digital Network (ISDN)

Integrated Services Digital Network is a set of communication standards for digital transmission of voice, video, data, and other network services over the traditional telephone network infrastructure. ISDN was developed to replace the old analog telephone system with a fully digital end to end communication network. It allows multiple digital signals to be transmitted simultaneously over a single telephone line, providing higher speed and better quality communication compared to analog systems.

The main purpose of ISDN is to provide a single unified digital network that can carry different types of communication services such as voice calls, video conferencing, data transfer, and fax simultaneously over the same line. It was designed to improve the speed, quality, and efficiency of communication by converting the entire telephone network from analog to digital transmission.


### Architecture of ISDN / Functional Group / Components

The architecture of ISDN is based on a digital end to end connection between the user and the network. It consists of several components. The **Terminal Equipment (TE)** is the user device such as a telephone or computer that connects to the ISDN network. The **Network Termination (NT)** is the device that connects the user premises to the ISDN network and handles the interface between the user side and the network side. The **Local Loop** is the digital connection between the user premises and the telephone exchange. The **ISDN Switch** is located at the telephone exchange and handles call routing and switching within the network.

- **TE1 (Terminal Equipment type 1)** is a device that is natively compatible with ISDN, such as an ISDN telephone or ISDN computer adapter, and can connect directly to the network without any conversion.
- **TE2 (Terminal Equipment type 2)** is a device that is not natively compatible with ISDN, such as a regular analog telephone or a standard computer, and requires a Terminal Adapter to connect to the ISDN network.
- **TA (Terminal Adapter)** is a device that connects non-ISDN equipment to the ISDN network by converting signals between the two formats, effectively acting as a bridge.
- **NT1 (Network Termination type 1)** connects the user premises to the ISDN local loop and handles the physical and electrical connection to the network.
- **NT2 (Network Termination type 2)** is a more intelligent device such as a Private Branch Exchange (PBX) that handles internal switching and distribution of ISDN channels within a building or organization.
- **ISDN Switch** is located at the telephone exchange on the network side and is responsible for call routing, switching, and managing connections between different users within the ISDN network.

### Interfaces

- **BRI (Basic Rate Interface)** uses a 2B + D configuration, meaning two Bearer channels for carrying user data and one Delta channel for signaling and control. It is designed for home users and small businesses with moderate communication needs.
- **PRI (Primary Rate Interface)** uses a higher capacity configuration with multiple Bearer channels and one Delta channel, designed for large businesses and organizations that require high volume communication. It comes in two variants:
  - **PRI Type 1 (North America/Japan)** — uses a 23B + D configuration, providing 23 channels for data and one for signaling.
  - **PRI Type 2 (Europe)** — uses a 30B + D configuration, providing 30 channels for data and one for signaling, offering slightly higher capacity.

### Channels

- **B Channel (Bearer channel)** is the primary data-carrying channel that transmits actual user content such as voice, video, and file transfers between endpoints at a fixed rate of 64 Kbps per channel.
- **D Channel (Delta channel)** does not carry user data but instead handles signaling, call setup, teardown, and control information; it operates at a lower speed in BRI and a higher speed in PRI, and can occasionally carry small amounts of packet data.
- **H Channel (High-speed channel)** is designed for bandwidth-intensive applications like video conferencing and high-speed data transfer, and comes in multiple variants (H0, H11, H12) each offering progressively higher capacity than a standard B channel.
