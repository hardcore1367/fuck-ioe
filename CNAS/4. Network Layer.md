# 4. Network Layer

The **Network Layer** is the third layer of the OSI model, responsible for end-to-end delivery of packets across multiple networks. Unlike the Data Link Layer which handles hop-to-hop delivery on a single link, the Network Layer provides logical addressing and routing so that data can travel across interconnected networks — this is the foundation of internetworking.

![Network Layer](https://github.com/user-attachments/assets/6f4aa078-506d-4f49-b8ae-3adaa9594cf9)

## 1. Functions of the Network Layer

### 1.1 Logical Addressing

Every device on a network is assigned a **logical address** (IP address) by the Network Layer. While MAC addresses identify devices within a single network, logical addresses uniquely identify every host globally, enabling packets to be delivered across multiple networks. The source and destination logical addresses are placed in the packet header and remain unchanged throughout the journey.

### 1.2 Routing

**Routing** is the process of determining the best path for a packet from source to destination across multiple networks. The Network Layer uses **routing algorithms** and **routing tables** stored in routers to make forwarding decisions at each hop. Routing can be static (manually configured) or dynamic (updated automatically using routing protocols like RIP or OSPF).

### 1.3 Internetworking

**Internetworking** is the ability to connect multiple heterogeneous networks so they function as a single unified network. The Network Layer provides this by abstracting the underlying physical and data link technologies — a packet can originate on an Ethernet network, pass through a token ring or Wi-Fi network, and arrive at the destination without the upper layers being aware of these transitions.

### 1.4 Fragmentation

**Fragmentation** occurs when a packet is too large to pass through a network whose **MTU (Maximum Transmission Unit)** is smaller than the packet size. The Network Layer breaks the original packet into smaller **fragments**, each with its own header containing an identification number, fragment offset, and a **More Fragments (MF)** flag. The destination reassembles all fragments into the original packet. If even one fragment is lost, the entire original packet must be retransmitted.

# 2. Networking devices

## 1. Repeater

A **repeater** operates at the **Physical Layer (Layer 1)**. Its sole function is to regenerate and amplify a weakened or distorted signal so it can travel further without degradation. As a signal travels through a cable, it loses strength due to attenuation — the repeater receives this weak signal, cleans it up, and retransmits it at full strength. It does not examine addresses, filter traffic, or make any forwarding decisions — it blindly retransmits every bit it receives on all ports. Because it works purely at the signal level, a repeater cannot connect networks using different protocols or speeds. It simply extends the physical range of a single network segment and all devices remain in the same collision domain.

![Repeater](https://github.com/user-attachments/assets/74c1fe7c-1661-4525-976d-e490ebdb9b9a)

## 2. Hub

A **hub** operates at the **Physical Layer (Layer 1)** and functions as a multiport repeater. When a frame arrives on any port, the hub broadcasts it out to every other port regardless of the intended destination. It has no awareness of MAC addresses or logical addressing — it treats the network as a single shared medium. As a result, all devices connected to a hub share the same **collision domain**, meaning only one device can transmit at a time and collisions are common under heavy load. Hubs also form a single **broadcast domain**, so broadcast traffic reaches every connected device. Because of these limitations, hubs are inefficient and have been almost entirely replaced by switches in modern networks.

![Hub](https://github.com/user-attachments/assets/3014f27d-be7b-4fa9-a6c0-65a4ee40e97b)

## 3. Bridge

A **bridge** operates at the **Data Link Layer (Layer 2)**. It connects two or more network segments and uses **MAC addresses** to make intelligent forwarding decisions. A bridge maintains a **MAC address table** which it builds dynamically by inspecting the source MAC address of every incoming frame — a process called **learning**. When a frame arrives, the bridge checks the destination MAC against its table. If the destination is on the same segment as the source, the frame is **filtered** (dropped) to avoid unnecessary traffic. If the destination is on a different segment, the frame is **forwarded** to that segment only. If the destination is unknown, the frame is **flooded** to all segments except the one it arrived from. A bridge splits the network into separate **collision domains** while all segments still belong to the same **broadcast domain**, so broadcast frames still reach all devices.

![Bridge](https://github.com/user-attachments/assets/ff259225-66f6-4647-a3b2-67bcb3f70049)

## 4. Switch

A **switch** operates at the **Data Link Layer (Layer 2)** and is essentially a multiport bridge with dedicated hardware for high-speed forwarding. It maintains a **MAC address table** (CAM table) that maps each known MAC address to the specific port it was learned from. When a frame arrives, the switch looks up the destination MAC and forwards the frame exclusively to the matching port — no other port receives that frame. This **unicast forwarding** eliminates the wasted bandwidth and collisions caused by hubs. Every port on a switch is its own **collision domain**, so multiple devices can transmit simultaneously without interfering with each other. Switches still forward broadcast frames to all ports, so all ports remain in the same **broadcast domain** unless VLANs are configured. Modern **Layer 3 switches** can also perform IP-based routing in addition to MAC-based switching.

![Switch](https://github.com/user-attachments/assets/df041be1-46f6-4f7d-999b-899953b14e60)

| Hub | Switch |
|---|---|
| Broadcasts to all ports regardless of destination. | Forwards only to the specific port matching the destination MAC. |
| All ports share one collision domain. | Each port is its own collision domain. |
| No MAC address table is maintained. | Maintains a MAC address table built dynamically by learning. |
| Suitable only for very small or legacy networks. | Preferred for all modern LAN connections. |

## 5. Router

A **router** operates at the **Network Layer (Layer 3)**. It connects multiple networks — including heterogeneous ones — and forwards packets based on **IP addresses** using a **routing table**. When a packet arrives, the router strips the frame header, examines the destination IP address in the packet header, consults its routing table to find the best next hop, re-encapsulates the packet in a new frame appropriate for the outgoing link, and transmits it. Unlike switches and bridges, a router separates both **collision domains** and **broadcast domains** — broadcast traffic is never forwarded between networks by a router. Routing tables are populated either manually (**static routing**) or automatically via **dynamic routing protocols** such as RIP, OSPF, or BGP. Routers are the primary devices responsible for internetworking — they are what allow packets to travel across the global internet by making hop-by-hop forwarding decisions at every intermediate network.

```
Layer 1 (Physical)  → Repeater, Hub
Layer 2 (Data Link) → Bridge, Switch

```

# 3. Internet Protocol

The **Internet Protocol (IP)** is the core Network Layer protocol responsible for logical addressing and routing packets across interconnected networks. IP is connectionless — it treats each packet independently and provides best-effort delivery with no guarantee of ordering, reliability, or error correction. These responsibilities are left to upper layers like TCP.

## 1. IP Address

An **IP address** is a numerical label assigned to every device on a network, used to identify the source and destination of packets. It has two parts — the **network portion** (identifies which network the device belongs to) and the **host portion** (identifies the specific device within that network). The boundary between these two parts is determined by the address class or subnet mask. Every packet carries both the source and destination IP address in its header, and routers use the destination IP to make forwarding decisions at each hop.

## 2. Classful Addressing

In **classful addressing**, the IPv4 address space is divided into five fixed classes — A, B, C, D, E — based on the leading bits of the address. The class an address belongs to automatically determines how many bits are used for the network portion and how many for the host portion, with no flexibility.

```
Class A: 0xxxxxxx   → large networks, few network IDs, many hosts
Class B: 10xxxxxx   → medium networks
Class C: 110xxxxx   → small networks, many network IDs, few hosts
Class D: 1110xxxx   → multicast
Class E: 1111xxxx   → experimental
```

## 3. Disadvantages of Classful Addressing

- **Address wastage** — An organization needing a few thousand hosts cannot fit in Class C (too small) and must be given Class B (far too large), leaving a massive portion of the block permanently unused.
- **Inflexibility** — The network/host boundary is rigidly fixed by class, so administrators have no way to adjust or subdivide their assigned block to match their actual network structure.
- **Accelerated address exhaustion** — Assigning oversized blocks to organizations regardless of actual need caused the IPv4 address space to deplete much faster than necessary.
- **Large routing tables** — Every classful network block requires its own separate entry in routing tables, causing routers to maintain increasingly large and slow tables as the internet grew.

## 4. Classless Addressing (CIDR)

**CIDR (Classless Inter-Domain Routing)** was developed to fix the limitations of classful addressing. Instead of fixed class boundaries, a **prefix length** written as `/n` is attached to the address to explicitly state how many bits belong to the network portion. This allows address blocks of any size to be allocated — not just the three fixed sizes of Class A, B, and C — so organizations get exactly the number of addresses they need without wastage.

```
Classful:  192.168.1.0      → always /24, fixed by class
Classless: 192.168.1.0/26  → smaller block, fewer hosts
           192.168.0.0/22  → larger block, spans multiple Class C ranges
```

CIDR also supports **supernetting** — combining multiple smaller blocks into a single larger block — which helps reduce the number of entries in routing tables.




## 4. IPv4

**IPv4 (Internet Protocol version 4)** is the fourth and most widely deployed version of IP. It uses **32-bit addresses** and is responsible for addressing and routing packets across networks. IPv4 is connectionless and unreliable — it delivers packets on a best-effort basis with no guarantee of delivery, ordering, or duplication prevention. These are handled by upper-layer protocols like TCP.

### 4.1 IPv4 Address Format

An IPv4 address is a 32-bit number written in **dotted decimal notation** — divided into four **octets** separated by dots, each ranging from 0 to 255.

```
Binary:  11000000 . 10101000 . 00000001 . 01011101
Decimal:    192   .   168   .    1     .    93
```

### 4.2 IPv4 Packet Header

```
 +---------+--------+------------+------------------+
 | Version |  IHL   |  DSCP/TOS  |   Total Length   |
 +---------+--------+------------+------------------+
 |      Identification      |  Flags | Frag. Offset  |
 +--------------------------+--------+---------------+
 |     TTL     |  Protocol  |    Header Checksum     |
 +-------------+------------+------------------------+
 |                 Source IP Address                  |
 +----------------------------------------------------+
 |              Destination IP Address                |
 +----------------------------------------------------+
 |              Options + Padding (if any)            |
 +----------------------------------------------------+
 |                       Data                         |
 +----------------------------------------------------+
```

### 4.3 Header Fields Explained

**1. Version** (4 bits) — Indicates the IP version in use. For IPv4, this value is always `4`. Routers use this field to determine how to interpret the rest of the header.

**2. IHL — Internet Header Length** (4 bits) — Specifies the length of the header in 32-bit words. The minimum value is 5 (meaning 20 bytes with no options). This field is needed because the Options field makes the header variable in length.

**3. DSCP/TOS — Type of Service** (8 bits) — Indicates the priority and handling preference for the packet. It is used for **Quality of Service (QoS)** — for example, voice traffic can be marked for low latency while bulk file transfers get best-effort handling.

**4. Total Length** (16 bits) — Specifies the total size of the packet in bytes, including both the header and the payload. The maximum value is 65,535 bytes.

**5. Identification** (16 bits) — A unique ID assigned to each original packet by the sender. When a packet is fragmented, all fragments of the same original packet share the same identification value so the destination can group them together for reassembly.

**6. Flags** (3 bits) — Controls fragmentation behavior. The **DF (Don't Fragment)** bit tells routers not to fragment the packet — if it is too large for a link, it is dropped and an ICMP error is sent back. The **MF (More Fragments)** bit is set on all fragments except the last one, signaling that more fragments are on the way.

**7. Fragment Offset** (13 bits) — Indicates the position of this fragment within the original unfragmented packet, measured in units of 8 bytes. The destination uses this to reassemble fragments in the correct order.

**8. TTL — Time To Live** (8 bits) — Limits how long a packet can circulate in the network. Every router that forwards the packet decrements TTL by 1. When TTL reaches 0, the router discards the packet and sends an **ICMP Time Exceeded** message back to the source, preventing packets from looping forever due to routing errors. Linux defaults to TTL 64; Windows uses 128.

**9. Protocol** (8 bits) — Identifies which Transport Layer protocol is carried in the payload, so the receiving host knows which process to deliver the data to after IP processing.

```
1  → ICMP
6  → TCP
17 → UDP
89 → OSPF
```

**10. Header Checksum** (16 bits) — Used to detect errors in the header only, not the payload. Each router recomputes the checksum after decrementing TTL and discards the packet if it does not match, ensuring the header has not been corrupted in transit.

**11. Source IP Address** (32 bits) — The IPv4 address of the sending device. Routers use this to send error messages back to the origin when needed.

**12. Destination IP Address** (32 bits) — The IPv4 address of the intended recipient. This is the primary field routers use to make forwarding decisions at every hop.

**13. Options + Padding** (variable) — Options is an optional field used for network testing, debugging, and special routing instructions such as source routing or record route. Since Options makes the header variable in length, **Padding** is added after it with zero bits to ensure the header ends on a 32-bit boundary. The IHL field tells routers exactly where the header ends.

**14. Data** — The actual payload delivered by IP — typically a TCP segment, UDP datagram, or ICMP message passed down from the Transport Layer.


# 5. Subnetting

**Subnetting** is the process of dividing a large network into smaller logical networks called **subnets**. This is done by borrowing bits from the host portion of the address and using them as additional network bits, effectively extending the prefix length. Each subnet behaves as an independent network with its own range of host addresses.

Subnetting is used when an organization has one large address block but needs to divide it into separate departments, floors, or locations — each isolated from the others. It reduces broadcast traffic, improves security, and makes networks easier to manage.

```
Original: 192.168.1.0/24  → 254 hosts, 1 network

Subnet into /26 (borrow 2 bits):
  192.168.1.0/26    → hosts: .1  to .62   (62 hosts)
  192.168.1.64/26   → hosts: .65 to .126  (62 hosts)
  192.168.1.128/26  → hosts: .129 to .190 (62 hosts)
  192.168.1.192/26  → hosts: .193 to .254 (62 hosts)
```

Each subnet has its own **network address** (first address, all host bits 0) and **broadcast address** (last address, all host bits 1), which are not assignable to hosts — hence 62 usable hosts per /26 subnet.

**VLSM (Variable Length Subnet Masking)** extends subnetting further by allowing different subnets within the same network to have different prefix lengths. This means a large department can get a /25 (126 hosts) while a small link between two routers gets a /30 (2 hosts), eliminating wastage entirely.

# 6. Supernetting

**Supernetting** is the opposite of subnetting — it combines multiple smaller contiguous network blocks into one larger block by moving the prefix boundary to the left, returning bits to the network portion. The result is a single route entry that represents all the combined networks.

Supernetting is primarily used by ISPs and backbone routers to reduce the number of routing table entries — a technique called **route aggregation** or **route summarization**. Instead of advertising hundreds of individual /24 networks, an ISP can advertise a single /20 that covers all of them.

```
Four Class C networks:
  192.168.0.0/24
  192.168.1.0/24
  192.168.2.0/24
  192.168.3.0/24

Supernet (aggregate):
  192.168.0.0/22  → covers all four blocks as one route entry
```

For supernetting to work, the blocks being combined must be **contiguous** and the starting address must be aligned to the boundary of the combined block size.

| Subnetting | Supernetting |
|---|---|
| Divides one large network into smaller subnets. | Combines multiple small networks into one larger block. |
| Prefix length increases (e.g., /24 → /26). | Prefix length decreases (e.g., /24 → /22). |
| Used by organizations to segment internal networks. | Used by ISPs to reduce routing table size. |
| Host bits are borrowed for network identification. | Network bits are returned to extend the block size. |
