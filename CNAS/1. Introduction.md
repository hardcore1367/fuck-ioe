# 1. Computer Network

A computer network is the infrastructure that allows computers and networking devices to exchange data. It is a collection of computers and other devices that use a common network protocol to share information with each other over a network medium.

## 1. Networking Model

### 1. Client Server Network

A client-server network is a model where two types of devices exist — clients and servers — each with a fixed role. The server is a powerful, always-on machine that stores data and provides services. The client is the device that requests those services. They never swap roles. A good example is when you open a website — your browser is the client, and the machine hosting that website is the server.

### 2. Peer to peer (P2P) Network

A peer-to-peer network is a model where every device (called a peer) acts as both a client and a server at the same time. There is no central machine in control — each peer shares its own resources directly with others. BitTorrent is a classic example, where users simultaneously download from and upload to other users.

## 2. Active Network

An active network is a network model where the intermediate nodes such as routers and switches are not just passing data packets from one point to another — they can actually process, modify, or even execute code inside those packets. Unlike traditional networks where nodes are passive and only forward data, active network nodes are programmable and can perform custom computations on the data as it travels through the network. A simple example would be a router that compresses or encrypts your data on the fly before forwarding it, without the sender or receiver needing to do anything extra.

## 3. Network Topology

Network topology refers to the arrangement or layout of devices (nodes) and connections (links) in a network. It defines how devices are physically or logically connected to each other.

## 4. Protocol and standard

A protocol is a set of rules that defines how data is transmitted and received between devices in a network. It ensures that two devices can communicate even if they are made by different manufacturers. Examples include HTTP, FTP, TCP, and IP.

A standard is a set of rules that has been officially approved and adopted by a recognized body so that different systems and devices can work together. Standards can be de facto (widely used in practice, like the QWERTY keyboard) or de jure (officially approved by organizations like ISO, IEEE, or ITU).

## 5. Layered Architecture

Layered architecture is a design approach where the communication system is broken down into distinct layers, each responsible for a specific task. Each layer provides services to the layer above it and uses services from the layer below it.

### Why Layered Architecture?

1. Breaks a complex networking problem into smaller, manageable pieces
2. Each layer can be developed, tested, and updated independently without affecting others
3. Allows different manufacturers to build compatible products as long as they follow the same layer rules
4. Makes troubleshooting easier since each layer has a defined responsibility
5. Promotes modularity and reusability across different network systems

### Design Issues

1. **Addressing** — Each layer needs a way to identify the sender and receiver at that level
2. **Error Control** — Detecting and correcting errors that occur during data transmission must be handled at appropriate layers
3. **Flow Control** — Ensuring a fast sender does not overwhelm a slow receiver with too much data at once
4. **Multiplexing** — Multiple conversations must be able to share the same physical connection simultaneously
5. **Routing** — When multiple paths exist, the network must decide which path the data should take
6. **Encapsulation** — Each layer adds its own header to the data before passing it down to the next layer

## 6. OSI Model

The OSI (Open Systems Interconnection) model is a conceptual framework developed by ISO that standardizes how different network systems communicate with each other. It divides the communication process into 7 distinct layers, where each layer has a specific responsibility and communicates only with the layers directly above and below it.

**Layer 7 - Application Layer**
This is the topmost layer and the one closest to the end user. It provides network services directly to user applications such as browsers, email clients, and file transfer programs. Protocols at this layer include HTTP, FTP, SMTP, and DNS.

**Layer 6 - Presentation Layer**
This layer is responsible for translating, encrypting, and compressing data so that the application layer can understand it. It acts as a translator between the network and the application, handling data format conversions such as JPEG, MP3, and encryption formats like SSL.

**Layer 5 - Session Layer**
This layer manages and controls the connections (sessions) between two communicating devices. It is responsible for establishing, maintaining, and terminating sessions. It also handles session recovery in case a connection drops in the middle of a transfer.

**Layer 4 - Transport Layer**
This layer is responsible for end-to-end communication and reliable data delivery between two devices. It breaks data into smaller segments, sends them, and reassembles them at the destination. It also handles error control and flow control. Key protocols here are TCP (reliable) and UDP (fast but unreliable).

**Layer 3 - Network Layer**
This layer is responsible for logical addressing and routing of data packets across multiple networks. It determines the best path for data to travel from source to destination. The most important protocol at this layer is IP (Internet Protocol). Routers operate at this layer.

**Layer 2 - Data Link Layer**
This layer is responsible for node-to-node delivery of data within the same network. It takes raw bits from the physical layer and organizes them into frames. It also handles error detection and MAC addressing. Switches and bridges operate at this layer.

**Layer 1 - Physical Layer**
This is the lowest layer and deals with the actual physical transmission of raw bits (0s and 1s) over a physical medium such as cables, fiber optics, or radio waves. It defines the hardware specifications like voltage levels, cable types, and data rates.

## 7. TCP/IP Model

The TCP/IP (Transmission Control Protocol/Internet Protocol) model is a practical, four-layer networking model that was developed by the US Department of Defense. Unlike the OSI model which is a theoretical framework, the TCP/IP model is the actual model used in the real world and forms the foundation of the modern internet. It is simpler than OSI as it combines several OSI layers into one.

**Layer 4 - Application Layer**
This is the topmost layer and combines the functions of OSI's application, presentation, and session layers into one. It provides protocols that applications use to communicate over the network. Common protocols include HTTP, FTP, SMTP, DNS, and Telnet.

**Layer 3 - Transport Layer**
This layer is responsible for end-to-end communication and reliable data delivery between two devices across the network. It breaks data into segments and ensures they are delivered correctly using two main protocols — TCP (Transmission Control Protocol) which is reliable and connection-oriented, and UDP (User Datagram Protocol) which is fast but connectionless.

**Layer 2 - Internet Layer**
This layer is responsible for logical addressing and routing of data packets across multiple networks. It determines the best path for data to reach its destination. The main protocol at this layer is IP (Internet Protocol). It also includes ICMP (for error reporting) and ARP (for address resolution).

**Layer 1 - Network Access Layer**
This is the lowest layer and combines the functions of OSI's data link and physical layers. It handles how data is physically transmitted over the network medium, including MAC addressing, framing, and the actual hardware transmission of bits over cables, fiber, or wireless signals.

| OSI Model | TCP/IP Model |
|---|---|
| Has 7 layers | Has 4 layers |
| Protocols are independent of the model, making it more flexible | Protocols are built into the model, making it less flexible |
| Each layer has a strict and clearly defined role and function | Layer boundaries are less strict and functions can overlap |
| Separate presentation layer handles encryption, compression and translation | No separate presentation layer, these functions are handled by application layer |
| Separate session layer manages session establishment and termination | No separate session layer, session management is done by application layer |
| Error control is performed at both data link and transport layer | Error control is primarily performed at transport layer only |
| Supports both connection-oriented and connectionless at network layer | Network layer (internet layer) is connectionless only |
| More reliable for large and complex enterprise networks | More reliable for real world internet communication |

## 8. Data Encapsulation in OSI Model

Data encapsulation is the process where each layer of the OSI model adds its own header (and sometimes trailer) to the data before passing it down to the next layer. At the receiving end, each layer removes its own header as the data moves up the layers. This process is called de-encapsulation.

### The Process

At the Application Layer, the user data is created. This raw data is then passed down to the Presentation Layer which formats, encrypts or compresses it and passes it further down.

At the Session Layer, session information is added to manage the communication and the data is passed to the Transport Layer. Here a header containing source and destination port numbers, sequence numbers and error control information is added. At this stage the data is called a Segment.

At the Network Layer, a header containing the source and destination IP addresses is added for routing purposes. At this stage the data is called a Packet.

At the Data Link Layer, both a header containing the MAC address and a trailer containing error detection information (CRC) is added. At this stage the data is called a Frame.

At the Physical Layer, the frame is converted into raw bits (0s and 1s) and transmitted over the physical medium such as cables or wireless signals.

| Layer | Data Name |
|---|---|
| Application, Presentation, Session | Data |
| Transport Layer | Segment |
| Network Layer | Packet |
| Data Link Layer | Frame |
