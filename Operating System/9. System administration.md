# System administration (4 hours) (10 marks)


## 1. System Administrator

A **system administrator (SA)** is a person responsible for managing, maintaining, and operating computer systems and networks within an organization. The SA ensures that all hardware, software, and network resources function correctly and securely for all users.

- Manages user accounts, permissions, and security policies
- Installs, configures, and updates operating systems and software
- Monitors system performance and resolves issues
- Maintains backups and ensures data recovery capability
- Enforces security policies and access controls


## 2. Roles and Duties of a System Administrator

A system administrator plays a central role in keeping an organization's IT infrastructure running smoothly. The SA is responsible for both day-to-day operations and long-term system planning.

- **User Management** — creates, modifies, and deletes user accounts; assigns permissions and group memberships. Example: creating a new account when an employee joins
- **Security Management** — sets up firewalls, manages passwords policies, monitors unauthorized access. Example: blocking an IP address after repeated failed logins
- **Backup and Recovery** — schedules regular backups and tests recovery procedures. Example: restoring a deleted database from last night's backup
- **Software Installation and Updates** — installs OS patches and application updates to fix vulnerabilities. Example: applying a kernel security patch across all servers
- **Performance Monitoring** — monitors CPU, memory, and disk usage and takes action when thresholds are exceeded. Example: identifying a runaway process consuming 100% CPU
- **Documentation** — maintains records of configurations, changes, and incidents for future reference


## 3. Role of System Administrator in Change Management

**Change management** is the process of controlling modifications to the system environment in a structured way to minimize disruption. The SA plays a key role in planning and executing changes safely.

- **Evaluating Changes** — reviews proposed changes to assess their impact on existing systems before approval
- **Planning and Scheduling** — schedules changes during low-usage periods to minimize disruption to users
- **Testing Changes** — tests changes in a staging or test environment before applying to production
- **Implementing Changes** — carries out the approved changes carefully, following documented procedures
- **Rollback Planning** — prepares a rollback plan so the system can be restored if a change causes failure
- **Communication** — informs affected users and stakeholders about upcoming changes and expected downtime
- **Post-change Review** — verifies that the change was successful and documents the outcome


## 4. Tasks of System Administrator in Operating System

The system administrator performs a wide range of tasks directly related to the operating system to keep it stable, secure, and efficient.

- **Installation and Configuration** — installs the OS and configures it for the organization's environment
- **User and Group Management** — creates user accounts, assigns them to groups, and manages permissions
- **File System Management** — manages disk partitions, mounts file systems, and monitors disk usage
- **Process Management** — monitors running processes, kills unresponsive processes, and manages priorities
- **Log Management** — reviews system logs (error logs, access logs) to detect problems or security issues
- **Scheduling Tasks** — uses tools like cron to automate repetitive tasks such as backups and cleanup scripts
- **Network Configuration** — configures IP addresses, DNS, routing, and firewall rules at the OS level
- **System Updates and Patches** — applies OS-level updates to fix bugs and security vulnerabilities


## 5. Key Technical and Soft Skills Needed by a System Administrator

A system administrator must possess a balanced mix of technical knowledge and interpersonal skills to be effective in the role.

**Technical Skills:**
- Knowledge of operating systems — Linux, Windows Server
- Networking fundamentals — TCP/IP, DNS, DHCP, firewalls
- Scripting and automation — Shell scripting, Python, AWK
- Security practices — access control, encryption, intrusion detection
- Backup and disaster recovery tools and procedures

**Soft Skills:**
- **Problem Solving** — ability to diagnose and resolve issues quickly under pressure
- **Communication** — explaining technical issues clearly to non-technical users and management
- **Time Management** — prioritizing multiple tasks and handling emergencies efficiently
- **Attention to Detail** — small misconfigurations can cause major system failures
- **Documentation** — keeping accurate records of all configurations and changes


## 6. Significance of System Administration

System administration is critical to the stability, security, and productivity of any organization that relies on computer systems.

- Ensures **continuous availability** of systems and services for all users
- Protects the organization against **security threats** and data breaches
- Reduces **downtime** through proactive monitoring and maintenance
- Enables **scalability** — SA prepares the infrastructure to handle growth
- Ensures **data integrity** through regular backups and recovery planning
- Saves costs by optimizing resource usage and preventing costly failures


## 7. AWK Tool

**AWK** is a powerful text-processing tool used in Unix/Linux for scanning and processing text files line by line. It is named after its creators — Aho, Weinberger, and Kernighan.

- Reads input line by line and splits each line into **fields** ($1, $2, $3...)
- Processes each line using **pattern-action** rules: `pattern { action }`
- Commonly used for filtering, formatting, and computing from text files

**Basic Syntax:**
```
awk 'pattern { action }' filename
```

**Examples:**
```bash
awk '{ print $1 }' file.txt        # print first field of every line
awk '$3 > 50 { print $1, $3 }' file.txt   # print lines where 3rd field > 50
awk '{ sum += $2 } END { print sum }' file.txt  # sum of 2nd column
```

- **BEGIN block** — runs before processing any line
- **END block** — runs after processing all lines
- Supports variables, loops, and conditional statements


## 8. Shell Programming

**Shell programming** (shell scripting) is the process of writing a sequence of shell commands in a file to automate tasks in Unix/Linux. The shell acts as an interpreter that executes these commands.

- Script files typically have **.sh** extension and begin with `#!/bin/bash` (shebang line)
- Used to automate repetitive tasks such as backups, user creation, and system monitoring

**Key Constructs:**

```bash
#!/bin/bash

# Variables
name="Prime"
echo "Hello $name"

# Input
read -p "Enter number: " num

# Conditionals
if [ $num -gt 10 ]; then
    echo "Greater than 10"
else
    echo "Not greater"
fi

# Loops
for i in 1 2 3; do
    echo $i
done

# While loop
while [ $i -le 5 ]; do
    echo $i
    i=$((i+1))
done
```

- Scripts are executed using `bash script.sh` or `./script.sh` (after `chmod +x`)
- Used extensively by system administrators for task automation and system management
