## Semaphore

A **semaphore** is an integer variable used for process synchronization, accessed only through two atomic operations — **wait()** and **signal()**. It prevents race conditions by ensuring mutual exclusion among concurrent processes. Proposed by **Dijkstra**.

## 1. Binary Semaphore

Value restricted to **0 or 1** only. Only one process can enter the critical section at a time.

**wait() and signal():**
```
wait(S):
  while S == 0
    do nothing;   // block until free
  S = 0;          // lock

signal(S):
  S = 1;          // unlock
```

**Usage:**
```
Semaphore mutex = 1;

wait(mutex);        // entry — sets to 0, others block
  // critical section
signal(mutex);      // exit — sets to 1, next process enters
```

## 2. Counting Semaphore

Value ranges from **0 to N**. Used when N instances of a resource are available.

**wait() and signal():**
```
wait(S):
  while S <= 0
    do nothing;   // block until a resource is free
  S--;            // acquire one resource

signal(S):
  S++;            // release one resource
```

**Usage:**
```
Semaphore resource = 3;   // 3 instances available

wait(resource);     // decrements count; blocks when count = 0
  // use resource
signal(resource);   // increments count; unblocks a waiting process
```

- When 3 processes are inside, 4th blocks at wait()
- When one exits and calls signal(), 4th is allowed in

## Comparison

| | Binary Semaphore | Counting Semaphore |
|---|---|---|
| Value | 0 or 1 | 0 to N |
| wait() blocks when | S == 0 | S <= 0 |
| signal() does | S = 1 | S++ |
| Use | Mutual exclusion | Resource counting |
| Processes allowed | 1 at a time | N at a time |

## Drawbacks

- **Busy waiting** — process wastes CPU looping in wait()
- **Deadlock** — if wait() and signal() are used in wrong order
- **Priority inversion** — low priority process holds semaphore needed by high priority process
