# 5. Project and Organizational Structure

An **organization** is a group of people with different knowledge and skills working together to achieve an organizational goal, and a project cannot run in isolation since it must always operate within this broader organizational environment. Because of this, a project manager needs to take a holistic, system-level view of the project and understand how it fits inside the larger organization, and the way that organization is structured directly affects how much authority and support the project manager actually has.

## 5.1 System View of Project Management

A **Project Management System** is an organized set of tools and procedures that helps managers and teams achieve project goals. This system view can be understood through three parts and three crucial elements.

### 5.1.1 Three Parts of the System View

- **Systems philosophy** views things as systems, meaning interacting components that work together within an environment to fulfill some purpose.
- **Systems analysis** is essentially a problem-solving approach applied to understand and resolve issues within the system.
- **Systems management** addresses business, technological, and organizational issues before making changes to the system.

### 5.1.2 Three Crucial Elements (Essences) of the System View

- A **structured approach**, represented by the organizational structure itself.
- A **disciplined approach**, represented by coordinated management processes.
- A **defined approach**, represented by purposefully defined inputs and outputs for each process.

### 5.1.3 Three-Sphere Model for Systems Management

![Three-Sphere Model for Systems Management](https://github.com/user-attachments/assets/37874dbc-c435-4f21-bf1c-0a8dade39733)


- The **business sphere** answers questions related to the business goals of the project.
- The **technological sphere** deals with resolving proper hardware and software issues.
- The **organizational sphere** covers issues involving the stakeholders of the project.

## 5.2 Organizational Structure

**Organizational structure** refers to the division of labor along with the patterns of coordination, communication, workflow, and formal power that direct an organization's activities. **Division of labor** refers to subdividing work into separate jobs assigned to different people, while **coordinating work activities** ensures that everyone works harmoniously toward the same goal despite this division. There are three main forms of organizational structure: functional, projectized (pure project), and matrix.

### 5.2.1 Functional Organization

A **functional organization** is a common structure in which the organization is divided into smaller groups based on specialization, and it is essentially an upgrade over line organization that overcomes its weaknesses by fully applying the principle of specialization. Activities are divided into functions such as production, marketing, and finance, and one specialist head oversees many workers according to functional authority.

![Functional Organization](https://github.com/user-attachments/assets/8544e7e8-2752-42c3-b7e8-a6cfd215317c)


- It groups employees by functional skill for a higher degree of performance.
- It increases the efficiency of workers and reduces the burden on top managers.
- It offers greater scope for training and development, and gives better control and supervision.
- It avoids duplication of work since each department is distinct with a clear job description.
- It violates the principle of unity of command, since workers report to many functional heads instead of one.
- It can slow down decision-making since functional managers must report further up to top authority.
- It can create conflicts between departments that encourage indiscipline in the organization.

### 5.2.2 Projectized (Pure Project) Organization

A **projectized organization**, also called a pure project organization, keeps the project completely separate from the functional units, unlike a functional organization. The project manager runs their own line organization with full project authority and responsibility, and projects report directly to the head of the organization, though this can create problems when coordinating with departmental functions.

![Projectized (Pure Project) Organization](https://github.com/user-attachments/assets/546afc60-ff69-4d87-9d1c-834c3b9419f7)


- It keeps sharp focus on project objectives with clear authority and effective communication.
- It allows flexible use of the labor force as required by the project.
- It causes duplication of facilities and effort across different projects.
- It offers less job security once a project ends, since staff are tied to that specific project.
- It can create conflict between project staff and non-project staff over project-specific benefits.
- It is generally useful only for large projects.

### 5.2.3 Matrix Organization

A **matrix organization** is a modern organization type established to face the challenges of a fast-changing market, and it mobilizes an organization's resources in a systematic manner. It provides reporting levels both horizontally and vertically, since staff report to both a functional manager and a project manager at the same time. Matrix organization is further classified into three types.

![Matrix Organization](https://github.com/user-attachments/assets/4e78a6f3-2e68-46fa-afa9-63246848e856)

- In a **strong matrix**, the project manager is more powerful in terms of authority, and although the project manager largely rules the project, team members still remain under their functional managers.
- In a **balanced matrix**, power is balanced between the project manager and the functional manager; staff are assigned to the project according to project needs by the project manager, while the functional manager facilitates all supporting resources.
- In a **weak matrix**, the project manager is simply a project coordinator with limited authority; the functional manager assigns the actual work, the project manager only negotiates for it, and mainly facilitates the project rather than directing it.

**Advantages of matrix organization:**

- It shares resources efficiently across projects.
- It keeps decision-making balanced and flexible.
- It lets staff communicate across departmental boundaries.
- It allows flexible use of resources and a fast response to change.
- It provides strong project coordination.

**Disadvantages of matrix organization:**

- It is costly to maintain since it needs both functional and project managers.
- It creates potential confusion over authority and responsibility.
- Its dual reporting structure adds confusion and can result in conflicts.
- Resources must be maintained throughout the project regardless of how long it takes.
- It creates issues whenever there is poor coordination between functional and project managers.

## 5.3 Organizational Structure Influences on Projects

Organizational structure represents the top-level view of the structure of ownership and the main line of reporting order for a project. The type of structure an organization uses directly influences how much authority a project manager actually holds; a project manager has the least authority in a functional organization, moderate authority in a matrix organization (varying by whether it is weak, balanced, or strong), and the most authority in a projectized organization. Even so, a project manager needs to follow a disciplined and definite approach to managing the project regardless of how much formal authority the structure gives them, since organizational structure only sets the boundaries within which that authority is exercised.
