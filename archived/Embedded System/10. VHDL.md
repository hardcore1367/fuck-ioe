## 1. VHDL Architecture Models

A **VHDL architecture** describes how a circuit behaves or is structured. There are three models:

**A) Behavioral** — Describes WHAT the circuit does using PROCESS blocks with sequential statements. Closest to software style.

**B) Dataflow** — Describes how data flows using concurrent signal assignments. Uses direct logic expressions, no PROCESS.

**C) Structural** — Describes the circuit as connected components. Uses COMPONENT and PORT MAP. Like drawing a schematic.

| Feature | Behavioral | Dataflow | Structural |
|---|---|---|---|
| Style | Sequential (PROCESS) | Concurrent expressions | Component connections |
| Uses PROCESS | Yes | No | No |
| Abstraction | High | Medium | Low |
| Like | Software code | Boolean equations | Circuit schematic |

---

## 2. Entity Declaration

**Entity** defines the input/output ports of a circuit — the interface (black box exterior). It does NOT describe behavior, only ports.

- `in` → input port
- `out` → output port
- `inout` → bidirectional

---

## 3. PROCESS in VHDL

A **PROCESS** block contains sequential statements that execute when any signal in the **sensitivity list** changes. Used in behavioral modeling.

- Sensitivity list — process triggers when listed signals change
- Statements inside execute **sequentially**
- Outside PROCESS, all statements are **concurrent**
- Used for: if-else logic, case statements, clocked circuits

---

## 4. COMPONENT Declaration

A **COMPONENT** is a reusable block inside a structural architecture. First declared, then instantiated using **PORT MAP**.

- Declaration must match the entity it refers to
- PORT MAP connects actual signals to component ports
- Allows building complex circuits from smaller reusable blocks
