# Circuit Design 

---

## 1. FPGA (Field Programmable Gate Array)

FPGA is a chip that can be programmed by the user after manufacturing. It has programmable logic blocks and interconnects. You program it using HDL languages like VHDL or Verilog. Unlike normal fixed chips, FPGA can be reprogrammed anytime.

**Key Points:**
- Contains programmable logic blocks + interconnects + I/O pins
- Programmed using VHDL or Verilog
- Config stored in SRAM or Flash
- Used for prototyping before making permanent chip (ASIC)
- Can be reprogrammed = very flexible
- Used in signal processing, communications, custom digital logic
- Faster to market, lower cost for small quantities

## 2. Reset and Power Failure Detection

Reset brings a circuit back to a known safe starting state when it starts up or crashes. Power Failure Detection monitors supply voltage and warns the system when voltage drops so it can save data before shutting down.

**Key Points — Reset:**
- Types: Power-on Reset (POR), Manual Reset, Watchdog Reset
- POR — auto resets when power is first applied
- Manual — user presses reset button
- Watchdog — resets when system hangs or freezes
- Ensures reliable and predictable startup

**Key Points — Power Failure Detection:**
- Continuously monitors supply voltage
- When voltage drops below threshold → triggers warning signal
- Allows system to save data before full power loss
- Uses voltage supervisor/monitor ICs
- Brown-out detection — detects partial voltage drop and resets system

## 3. Impedance Matching

Impedance matching means making the impedance of source, transmission line, and load all equal. At high frequencies, if impedances are not matched, signals don't transfer properly and cause serious problems like reflections and distortion.

**Key Points — Need:**
- To maximize power transfer
- To prevent signal reflections
- To maintain signal integrity at high frequencies

**Key Points — Impact of Impedance Discontinuities:**
- Signal reflections — part of signal bounces back
- Signal distortion and reduced amplitude
- Ringing (unwanted oscillations) on signals
- Increased EMI

**Key Points — Stub Discontinuity:**
- Stub is an unterminated branch on a transmission line
- Creates impedance mismatch at junction
- Causes resonances at certain frequencies
- Creates notches (dips) in frequency response
- Should be minimized or properly terminated

## 4. Converting Requirements into Circuit Design

This is the process of taking project requirements and turning them into an actual working circuit. It involves understanding the problem, planning, selecting components, and building and testing step by step.

**Key Points — General Approach:**
1. Understand and analyze all requirements
2. Break system into functional blocks
3. Select appropriate components
4. Draw schematic/circuit diagram
5. Simulate and verify
6. Build prototype and test
7. Refine and iterate

**Key Points — Tips:**
- Always start with a block diagram
- Consider worst-case conditions
- Use standard/proven circuits where possible
- Document all design decisions
- Consider cost, availability, and manufacturability

**Key Points — Design Dilemmas (Trade-offs):**
- Performance vs power consumption
- Cost vs reliability
- Speed vs noise
- Size vs heat dissipation
- Complexity vs simplicity

## 5. Low Power Design

Low power design means designing circuits that consume as little power as possible. This is important for battery-operated devices, reducing heat, and lowering operating costs.

**Key Points — Reasons:**
- Longer battery life in portable devices
- Less heat, no need for extra cooling
- Lower operating costs
- Environmental/green concerns
- Enables smaller and lighter designs

**Key Points — Guidelines:**
- Reduce supply voltage as much as possible
- Clock gating — turn off clock when not in use
- Power gating — turn off unused circuit blocks
- Use low-power components
- Minimize unnecessary switching activity

**Key Points — Techniques:**
- DVFS (Dynamic Voltage and Frequency Scaling) — reduce voltage/frequency when full speed not needed
- Sleep/Standby modes — put system to sleep when idle
- Use efficient algorithms to reduce computation
- Multiple voltage domains — different parts run at different voltages
- Use CMOS logic family (naturally low power)

## 6. Reliability in Circuit Design

Reliability is the probability that a circuit will perform its function without failure for a specified time under specified conditions. A reliable circuit keeps working correctly for a long time without breaking down.

**Key Points — Factors Affecting Reliability:**
- Temperature — higher temp = lower reliability (most critical factor)
- Voltage stress — running near max limits reduces life
- Humidity and moisture
- Mechanical stress and vibration
- Component quality
- Manufacturing defects
- Aging and wear-out over time

**Key Points — Measures to Achieve Reliability:**
- Use quality/rated components
- Derating — operate below maximum rated values
- Proper thermal management (heatsinks, ventilation)
- Environmental protection (conformal coating on PCB)
- Add redundancy in critical parts
- Burn-in testing — stress test before deployment
- Use FMEA (Failure Mode and Effects Analysis) to predict failures
- Track MTBF (Mean Time Between Failures) as reliability measure

## 7. High Speed Design

High speed design deals with circuits that operate at very high frequencies. At high speeds, things like signal reflections, noise, and interference become serious problems that must be carefully managed during design.

**Key Points — Main Concerns:**
- Signal reflections on PCB traces
- Crosstalk between nearby traces
- Electromagnetic interference (EMI)
- Power supply noise
- Propagation delays

**Key Points — Guidelines:**
- Keep signal traces short
- Use controlled impedance PCB traces (e.g., 50Ω)
- Proper termination at end of traces
- Use ground planes under signal traces
- Minimize use of vias (they add discontinuities)
- Place decoupling capacitors close to ICs
- Use differential signaling for noise immunity

## 8. Fault Tolerance Schemes

Fault tolerance is the ability of a circuit or system to continue working correctly even when some of its parts fail. Instead of completely stopping on failure, a fault tolerant system handles the fault and keeps running.

**Key Points — Redundancy:**
- Having backup/duplicate components
- If one fails, backup takes over automatically
- Hardware redundancy — duplicate circuits
- Software redundancy — backup code/logic
- Time redundancy — repeat operation and compare results

**Key Points — Graceful Degradation:**
- When part of system fails, system continues at reduced performance instead of fully stopping
- Example: multi-core processor working with fewer cores if one fails

**Key Points — Watchdog Timer:**
- A timer that must be reset periodically by the running system
- If system hangs or crashes, timer expires and automatically resets/recovers the system
- Simple and effective way to detect system lockups

**Key Points — Rollback:**
- When fault is detected, system goes back to a previous known-good state
- System saves checkpoints during normal operation
- On failure, restores from last saved checkpoint

**Key Points — Factors Affecting Reliability:**
- Temperature, voltage stress, humidity, component quality, mechanical stress, aging, manufacturing defects
